## Dev Notes

### App.xaml.cs Bootstrapping Pattern

The application uses `Microsoft.Extensions.Hosting.GenericHost` to bootstrap DI. This is the recommended pattern for WPF apps with full DI support:

```csharp
public partial class App : Application
{
    private IHost? _host;

    protected override async void OnStartup(StartupEventArgs e)
    {
        base.OnStartup(e);

        _host = Host.CreateDefaultBuilder()
            .UseSerilog((context, services, configuration) =>
            {
                configuration
                    .MinimumLevel.Information()
                    .MinimumLevel.Override("Microsoft", LogEventLevel.Warning)
                    .WriteTo.Console()
                    .WriteTo.File("logs/dspanel-.log",
                        rollingInterval: RollingInterval.Day,
                        retainedFileCountLimit: 30);
            })
            .ConfigureServices((context, services) =>
            {
                // ViewModels
                services.AddTransient<MainViewModel>();

                // Windows
                services.AddSingleton<MainWindow>();
            })
            .Build();

        await _host.StartAsync();

        var mainWindow = _host.Services.GetRequiredService<MainWindow>();
        mainWindow.Show();
    }

    protected override async void OnExit(ExitEventArgs e)
    {
        if (_host is not null)
        {
            await _host.StopAsync();
            _host.Dispose();
        }
        base.OnExit(e);
    }
}
```

### MainViewModel Skeleton

```csharp
using CommunityToolkit.Mvvm.ComponentModel;

namespace DSPanel.ViewModels;

public partial class MainViewModel : ObservableObject
{
    [ObservableProperty]
    private string _title = "DSPanel";
}
```

The `[ObservableProperty]` attribute triggers the source generator to produce a public `Title` property with `INotifyPropertyChanged` support.

### Source Tree Conventions

Every public class in `src/DSPanel/` that contains logic should have a corresponding test class in `src/DSPanel.Tests/` following the mirror convention:

- `src/DSPanel/Services/Foo.cs` - `src/DSPanel.Tests/Services/FooTests.cs`
- `src/DSPanel/ViewModels/MainViewModel.cs` - `src/DSPanel.Tests/ViewModels/MainViewModelTests.cs`

### Testing

- Framework: xUnit 2.9.3 with `[Fact]` and `[Theory]` attributes
- Mocking: Moq 4.20.72 for interface mocking
- Assertions: FluentAssertions 8.8.0 (`result.Should().Be(...)`)
- Bootstrap tests should verify the DI container builds correctly and key services resolve
- Tests must not require a running WPF dispatcher - use unit-testable patterns only
- Test naming convention: `MethodName_Scenario_ExpectedResult`
