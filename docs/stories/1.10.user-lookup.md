# Story 1.10: User Account Lookup

## Status: Draft

## Story

**As a** L1 support technician,
**I want** to search for a user account and see their complete information,
**so that** I can quickly diagnose account issues.

## Acceptance Criteria

1. Search bar accepts SAMAccountName, UPN, display name, or partial match and triggers search on Enter or button click.
2. Results list displays matching users with name, department, and account status (enabled/disabled/locked).
3. Detail view shows identity fields: display name, first name, last name, email, department, title, OU breadcrumb, and distinguished name.
4. Detail view shows account status: enabled/disabled, locked/unlocked indicators.
5. Detail view shows authentication info: failed password count, last logon timestamp, last logon workstation and IP address.
6. Detail view shows group memberships in a sortable and filterable DataTable.
7. All LDAP queries are async and the UI shows a loading spinner during search.
8. Empty state is displayed when no results match the query.
9. Error state is displayed with a retry option when the LDAP query fails.

## Dev Notes

### Architecture

UserLookup follows a React component + Tauri command pattern. The page component manages local state and calls Tauri backend commands for LDAP operations.

### DirectoryUser Type

The `DirectoryUser` type in `src/types/directory.ts` contains 28+ fields mapped from LDAP attributes:

| Property | LDAP Attribute | Type |
|----------|---------------|------|
| samAccountName | sAMAccountName | string |
| userPrincipalName | userPrincipalName | string |
| displayName | displayName | string |
| givenName | givenName | string |
| surname | sn | string |
| email | mail | string |
| department | department | string |
| title | title | string |
| distinguishedName | distinguishedName | string |
| organizationalUnit | (parsed from DN) | string |
| enabled | userAccountControl | boolean |
| lockedOut | lockoutTime | boolean |
| accountExpires | accountExpires | string (ISO date) or null |
| passwordLastSet | pwdLastSet | string (ISO date) or null |
| passwordExpired | (computed) | boolean |
| passwordNeverExpires | userAccountControl | boolean |
| lastLogon | lastLogon | string (ISO date) or null |
| lastLogonWorkstation | (operational) | string |
| badPasswordCount | badPwdCount | number |
| whenCreated | whenCreated | string (ISO date) |
| whenChanged | whenChanged | string (ISO date) |
| memberOf | memberOf | string[] |

### Component Pattern

```typescript
// src/pages/UserLookup.tsx
import { useState, useCallback } from "react";
import { invoke } from "@tauri-apps/api/core";

export function UserLookup() {
  const [searchQuery, setSearchQuery] = useState("");
  const [isLoading, setIsLoading] = useState(false);
  const [selectedUser, setSelectedUser] = useState<DirectoryUser | null>(null);
  const [searchResults, setSearchResults] = useState<DirectoryUser[]>([]);
  const [error, setError] = useState<string | null>(null);

  const handleSearch = useCallback(async () => {
    setIsLoading(true);
    setError(null);
    try {
      const results = await invoke<DirectoryUser[]>("search_users", {
        query: searchQuery,
      });
      setSearchResults(results);
    } catch (err) {
      setError(err instanceof Error ? err.message : "Search failed");
    } finally {
      setIsLoading(false);
    }
  }, [searchQuery]);

  // ... render
}
```

### Rust Tauri Command

```rust
// src-tauri/src/commands/user.rs
use tauri::State;
use crate::state::AppState;
use crate::models::DirectoryUser;

#[tauri::command]
pub async fn search_users(
    query: String,
    state: State<'_, AppState>,
) -> Result<Vec<DirectoryUser>, String> {
    let provider = state.directory_provider.lock().map_err(|e| e.to_string())?;
    provider
        .search_users(&query, 50)
        .await
        .map_err(|e| e.to_string())
}
```

### OU Breadcrumb Parsing

The OU breadcrumb is parsed from the DistinguishedName by extracting `OU=` components in reverse order:

```
CN=John Doe,OU=Engineering,OU=Departments,DC=corp,DC=local
-> Departments > Engineering
```
