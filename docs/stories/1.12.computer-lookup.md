# Story 1.12: Computer Account Lookup

## Status: Done

## Story

**As a** support technician,
**I want** to search for a computer account and see its details,
**so that** I can diagnose workstation issues.

## Acceptance Criteria

1. Search bar accepts computer name with exact and partial matching.
2. Detail view displays: computer name, DNS hostname, operating system, OS version, last logon timestamp, OU breadcrumb, and enabled/disabled status.
3. Group memberships are displayed in a sortable and filterable DataTable.
4. Ping button sends an ICMP ping and displays reachable/unreachable status with latency in milliseconds.
5. DNS resolution displays the resolved IP address(es) for the computer's DNS hostname.
6. Computer lookup uses the same tab system as user lookups for consistent navigation.

## Dev Notes

### Architecture

ComputerLookup follows the same pattern as UserLookup. The page layout mirrors UserLookup for consistency. Network operations (ping, DNS) are Rust Tauri commands that run asynchronously.

### Rust DirectoryComputer Model

```rust
// src-tauri/src/models/directory_computer.rs
use serde::{Deserialize, Serialize};

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct DirectoryComputer {
    pub name: String,
    pub dns_host_name: String,
    pub operating_system: String,
    pub os_version: String,
    pub last_logon: Option<String>,
    pub distinguished_name: String,
    pub organizational_unit: String,
    pub enabled: bool,
    pub member_of: Vec<String>,
}
```

LDAP attribute mapping:

| Property | LDAP Attribute |
|----------|---------------|
| name | cn |
| dns_host_name | dNSHostName |
| operating_system | operatingSystem |
| os_version | operatingSystemVersion |
| last_logon | lastLogon |
| distinguished_name | distinguishedName |
| organizational_unit | (parsed from DN) |
| enabled | userAccountControl (bit flag 0x0002 = ACCOUNTDISABLE) |
| member_of | memberOf |

### Ping Tauri Command

```rust
// src-tauri/src/commands/network.rs
use std::time::Duration;

#[tauri::command]
pub async fn ping_host(hostname: String) -> Result<String, String> {
    // Using surge-ping or similar crate
    match ping_with_timeout(&hostname, Duration::from_millis(3000)).await {
        Ok(latency) => Ok(format!("Reachable ({}ms)", latency.as_millis())),
        Err(_) => Ok("Unreachable".to_string()),
    }
}
```

### DNS Resolution Tauri Command

```rust
#[tauri::command]
pub async fn resolve_dns(hostname: String) -> Result<Vec<String>, String> {
    match tokio::net::lookup_host(format!("{}:0", hostname)).await {
        Ok(addrs) => Ok(addrs.map(|a| a.ip().to_string()).collect()),
        Err(_) => Err("DNS resolution failed".to_string()),
    }
}
```

DNS resolution is triggered automatically from the frontend when `selectedComputer` changes via `useEffect`.

### Frontend Component

```typescript
// src/pages/ComputerLookup.tsx
useEffect(() => {
  if (selectedComputer) {
    invoke<string[]>("resolve_dns", { hostname: selectedComputer.dnsHostName })
      .then(setResolvedAddresses)
      .catch(() => setResolvedAddresses(["DNS resolution failed"]));
  }
}, [selectedComputer]);

const handlePing = async () => {
  if (!selectedComputer) return;
  setIsPinging(true);
  try {
    const result = await invoke<string>("ping_host", {
      hostname: selectedComputer.dnsHostName,
    });
    setPingResult(result);
  } catch {
    setPingResult("Unreachable (ping failed)");
  } finally {
    setIsPinging(false);
  }
};
```
