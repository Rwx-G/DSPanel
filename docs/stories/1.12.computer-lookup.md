# Story 1.12: Computer Account Lookup

## Status: Done

## Story

**As a** support technician,
**I want** to search for a computer account and see its details,
**so that** I can diagnose workstation issues.

## Acceptance Criteria

1. Search bar accepts computer name with exact and partial matching.
2. Detail view displays: computer name, DNS hostname, operating system, OS version, last logon timestamp, OU breadcrumb, and enabled/disabled status.
3. Group memberships are displayed in a sortable and filterable DataGrid.
4. Ping button sends an ICMP ping and displays reachable/unreachable status with latency in milliseconds.
5. DNS resolution displays the resolved IP address(es) for the computer's DNS hostname.
6. Computer lookup uses the same tab system as user lookups for consistent navigation.

## Dev Notes

### Architecture

ComputerLookupViewModel follows the same pattern as UserLookupViewModel. The view layout mirrors UserLookupView for consistency. Network operations (ping, DNS) are wrapped in try/catch blocks and run asynchronously.

### DirectoryComputer Model

```csharp
public class DirectoryComputer
{
    public string Name { get; init; } = string.Empty;
    public string DnsHostName { get; init; } = string.Empty;
    public string OperatingSystem { get; init; } = string.Empty;
    public string OperatingSystemVersion { get; init; } = string.Empty;
    public DateTime? LastLogon { get; init; }
    public string DistinguishedName { get; init; } = string.Empty;
    public string OrganizationalUnit { get; init; } = string.Empty;
    public bool Enabled { get; init; }
    public IReadOnlyList<string> MemberOf { get; init; } = [];
}
```

LDAP attribute mapping:

| Property | LDAP Attribute |
|----------|---------------|
| Name | cn |
| DnsHostName | dNSHostName |
| OperatingSystem | operatingSystem |
| OperatingSystemVersion | operatingSystemVersion |
| LastLogon | lastLogon |
| DistinguishedName | distinguishedName |
| OrganizationalUnit | (parsed from DN) |
| Enabled | userAccountControl (bit flag 0x0002 = ACCOUNTDISABLE) |
| MemberOf | memberOf |

### Ping Implementation

```csharp
[RelayCommand]
private async Task PingAsync()
{
    if (SelectedComputer is null) return;
    IsPinging = true;
    try
    {
        using var ping = new System.Net.NetworkInformation.Ping();
        var reply = await ping.SendPingAsync(SelectedComputer.DnsHostName, 3000);
        PingResult = reply.Status == IPStatus.Success
            ? $"Reachable ({reply.RoundtripTime}ms)"
            : "Unreachable";
    }
    catch (PingException)
    {
        PingResult = "Unreachable (ping failed)";
    }
    finally
    {
        IsPinging = false;
    }
}
```

### DNS Resolution

```csharp
[RelayCommand]
private async Task ResolveDnsAsync()
{
    if (SelectedComputer is null) return;
    try
    {
        var addresses = await Dns.GetHostAddressesAsync(SelectedComputer.DnsHostName);
        ResolvedAddresses = string.Join(", ", addresses.Select(a => a.ToString()));
    }
    catch (SocketException)
    {
        ResolvedAddresses = "DNS resolution failed";
    }
}
```

DNS resolution is triggered automatically when `SelectedComputer` changes via the `OnSelectedComputerChanged` partial method.
