# Story 1.2: Directory Provider and AD On-Prem Connection

## Status: Done

## Story

**As a** developer,
**I want** a `DirectoryProvider` trait with an LDAP implementation,
**so that** all AD queries go through a consistent abstraction.

## Acceptance Criteria

1. `DirectoryProvider` trait defines methods for searching users, computers, and groups.
2. `LdapDirectoryProvider` implements `DirectoryProvider` using the `ldap3` crate.
3. Domain base DN is auto-detected via the `USERDNSDOMAIN` environment variable and confirmed with a `rootDSE` query.
4. Authentication uses the current user's Kerberos credentials (no stored passwords).
5. `LdapDirectoryProvider` is registered in Tauri state as a shared singleton.
6. Unit tests cover all `DirectoryProvider` methods using mock LDAP dependencies.
7. Graceful error handling when no domain is available (non-domain-joined machine) - returns empty results and logs a warning.

### LDAP Connection Pattern

```rust
use ldap3::{LdapConnAsync, LdapConnSettings, Scope, SearchEntry};

// Domain detection
let domain = std::env::var("USERDNSDOMAIN").ok();
let Some(domain) = domain else {
    tracing::warn!("USERDNSDOMAIN not set - not domain-joined");
    return Ok(());
};

// Connect with Kerberos (GSSAPI)
let settings = LdapConnSettings::new();
let (conn, mut ldap) = LdapConnAsync::with_settings(
    settings,
    &format!("ldap://{}:389", domain),
).await?;
tokio::spawn(conn.drive());
ldap.sasl_gssapi_bind(&domain).await?;

// Discover base DN via rootDSE
let (rs, _) = ldap.search(
    "",
    Scope::Base,
    "(objectClass=*)",
    vec!["defaultNamingContext"],
).await?.success()?;

let base_dn = rs.into_iter()
    .next()
    .and_then(|entry| {
        let se = SearchEntry::construct(entry);
        se.attrs.get("defaultNamingContext")
            .and_then(|v| v.first().cloned())
    });
```

### LDAP Search Pattern

```rust
let filter = format!(
    "(&(objectClass=user)(objectCategory=person)\
     (|(sAMAccountName=*{}*)(displayName=*{}*)))",
    ldap3::ldap_escape(filter),
    ldap3::ldap_escape(filter),
);

let (rs, _) = ldap.search(
    &base_dn,
    Scope::Subtree,
    &filter,
    vec![
        "distinguishedName", "sAMAccountName", "displayName",
        "objectClass", "mail", "department", "title", "telephoneNumber",
    ],
).await?.success()?;
```

### Important Notes

- `ldap3::LdapConn` is not `Clone`. Use `Arc<Mutex<Ldap>>` or create connections per request.
- LDAP filters must escape special characters: `*`, `(`, `)`, `\`, NUL. Use `ldap3::ldap_escape()`.
- The provider should be registered as a singleton in Tauri state but create connections lazily.
- All operations are async via `tokio`.
