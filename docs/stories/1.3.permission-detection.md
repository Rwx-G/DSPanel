# Story 1.3: Permission Level Detection

## Status: Done

## Story

**As a** support technician,
**I want** DSPanel to detect my permission level from AD groups,
**so that** I only see authorized actions.

## Acceptance Criteria

1. `PermissionService` detects the current user's AD group memberships at startup.
2. Group memberships are mapped to a `PermissionLevel` enum: `ReadOnly`, `HelpDesk`, `AccountOperator`, `DomainAdmin`.
3. AD group names used for detection are configurable (not hardcoded).
4. Permission levels follow inheritance - a `DomainAdmin` implicitly has all lower-level permissions.
5. The current permission level is displayed in the status bar.
6. `has_permission(required: PermissionLevel) -> bool` returns `true` if the user meets or exceeds the required level.
7. Unit tests cover all permission detection and inheritance scenarios.

## Dev Notes

### PermissionLevel Enum

```rust
// src-tauri/src/services/permissions.rs
use serde::{Deserialize, Serialize};

#[derive(Debug, Clone, Copy, PartialEq, Eq, PartialOrd, Ord, Serialize, Deserialize)]
#[repr(u8)]
pub enum PermissionLevel {
    ReadOnly = 0,
    HelpDesk = 1,
    AccountOperator = 2,
    DomainAdmin = 3,
}
```

The derived `PartialOrd` and `Ord` enable simple comparison for inheritance: `current_level >= required_level`.

### Default Group Name Mappings

| PermissionLevel | Default AD Group Name |
|----------------|----------------------|
| HelpDesk | DSPanel-HelpDesk |
| AccountOperator | DSPanel-AccountOps |
| DomainAdmin | Domain Admins |

`ReadOnly` is the default level assigned when no matching groups are found. There is no specific AD group for it.

### PermissionService Implementation

```rust
use std::collections::HashMap;
use std::sync::Mutex;

pub struct PermissionService {
    current_level: Mutex<PermissionLevel>,
    user_groups: Mutex<Vec<String>>,
    group_mappings: HashMap<String, PermissionLevel>,
}

impl PermissionService {
    pub fn has_permission(&self, required: PermissionLevel) -> bool {
        *self.current_level.lock().unwrap() >= required
    }
}
```

### Inheritance Logic

The service scans all group memberships and assigns the highest matching level:

```rust
pub async fn detect_permissions(&self, provider: &dyn DirectoryProvider) -> Result<()> {
    let user_groups = provider.get_current_user_groups().await?;
    let mut detected_level = PermissionLevel::ReadOnly;

    for group in &user_groups {
        if let Some(&level) = self.group_mappings.get(group) {
            if level > detected_level {
                detected_level = level;
            }
        }
    }

    *self.current_level.lock().unwrap() = detected_level;
    *self.user_groups.lock().unwrap() = user_groups;
    Ok(())
}
```

### Configuration Pattern

Group-to-level mappings are loaded from a configuration file so they can be customized per deployment:

```json
{
  "permissions": {
    "group_mappings": {
      "DSPanel-HelpDesk": "HelpDesk",
      "DSPanel-AccountOps": "AccountOperator",
      "Domain Admins": "DomainAdmin"
    }
  }
}
```

Use `serde_json` to deserialize the configuration into a `PermissionConfig` struct.

### Querying User Groups

The service uses `DirectoryProvider` to fetch the current user's group memberships. The current user's `sAMAccountName` is obtained from `std::env::var("USERNAME")`. Groups are read from the `memberOf` attribute of the user object.

### Tauri State Registration

```rust
// In main.rs setup
let permission_service = PermissionService::new(config);
app.manage(permission_service);
```

`detect_permissions()` is called during application startup after the `DirectoryProvider` connection is established.

### Frontend Hook

```typescript
// src/hooks/usePermissions.ts
import { invoke } from "@tauri-apps/api/core";

export function usePermissions() {
  const [level, setLevel] = useState<PermissionLevel>("ReadOnly");

  useEffect(() => {
    invoke<PermissionLevel>("get_permission_level").then(setLevel);
  }, []);

  const hasPermission = (required: PermissionLevel) => {
    const levels = ["ReadOnly", "HelpDesk", "AccountOperator", "DomainAdmin"];
    return levels.indexOf(level) >= levels.indexOf(required);
  };

  return { level, hasPermission };
}
```
