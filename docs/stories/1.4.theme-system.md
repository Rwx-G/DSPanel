# Story 1.4: Theme System, Design Tokens and Style Architecture

## Status: Draft

## Story

**As a** developer,
**I want** a comprehensive theme system with design tokens, dark/light modes, and organized CSS architecture,
**so that** the app has a consistent visual identity.

## Acceptance Criteria

1. Color palette is defined as CSS custom properties (design tokens) in dedicated CSS files.
2. Dark and light themes are switchable at runtime without restarting the application.
3. Typography scale defines heading levels 1-4, body, caption, and mono styles.
4. A 4px-based spacing scale provides consistent margins and padding values.
5. Border radius, shadow, and elevation tokens are defined for layered UI depth.
6. Icon system uses SVG components or an icon library for scalable icons.
7. All base component styles are defined: Button, Input, Select, Checkbox, Table, Tabs, List, Scrollbar.
8. Theme preference is persisted and restored on next launch.
9. Unit tests verify that theme switching works correctly and all tokens are defined.

## Dev Notes

### CSS Architecture

The `src/styles/` directory organizes styles in this order:

```
src/styles/
  tokens.css          # Base design tokens (spacing, typography, elevation)
  theme-light.css     # Light theme color values
  theme-dark.css      # Dark theme color values
  components.css      # Base component styles
  utilities.css       # Utility classes
```

Import order in `src/main.tsx`:
```typescript
import "./styles/tokens.css";
import "./styles/theme-light.css";
import "./styles/theme-dark.css";
import "./styles/components.css";
```

### Runtime Theme Switching

Themes are switched by setting a `data-theme` attribute on the root `<html>` element:

```typescript
// src/hooks/useTheme.ts
import { useState, useEffect, useCallback } from "react";

type ThemeMode = "light" | "dark";

export function useTheme() {
  const [currentTheme, setCurrentTheme] = useState<ThemeMode>(() => {
    const saved = localStorage.getItem("dspanel-theme");
    if (saved === "light" || saved === "dark") return saved;
    return window.matchMedia("(prefers-color-scheme: dark)").matches
      ? "dark"
      : "light";
  });

  const applyTheme = useCallback((mode: ThemeMode) => {
    document.documentElement.setAttribute("data-theme", mode);
    localStorage.setItem("dspanel-theme", mode);
    setCurrentTheme(mode);
  }, []);

  useEffect(() => {
    applyTheme(currentTheme);
  }, []);

  return { currentTheme, applyTheme };
}
```

### Semantic Color Token Names

All color tokens use CSS custom properties scoped to the `data-theme` attribute:

```css
/* theme-light.css */
[data-theme="light"] {
  --color-primary: #1976d2;
  --color-primary-hover: #1565c0;
  --color-primary-pressed: #0d47a1;
  --color-surface-bg: #ffffff;
  --color-surface-card: #f5f5f5;
  --color-surface-elevated: #ffffff;
  --color-text-primary: #212121;
  --color-text-secondary: #757575;
  --color-text-disabled: #bdbdbd;
  --color-border-default: #e0e0e0;
  --color-border-strong: #9e9e9e;
  --color-success: #4caf50;
  --color-warning: #ff9800;
  --color-error: #f44336;
  --color-info: #2196f3;
}
```

### Icon System

Using `lucide-react` or inline SVG components:

```typescript
// src/components/Icon.tsx
import { LucideIcon } from "lucide-react";

interface IconProps {
  icon: LucideIcon;
  size?: number;
  className?: string;
}

export function Icon({ icon: IconComponent, size = 16, className }: IconProps) {
  return <IconComponent size={size} className={className} />;
}
```

### Button Component Style Pattern

```css
.btn-primary {
  background-color: var(--color-primary);
  color: var(--color-text-inverse);
  font-size: 14px;
  padding: var(--space-sm) var(--space-lg);
  border: none;
  border-radius: var(--radius-md);
  cursor: pointer;
}

.btn-primary:hover {
  background-color: var(--color-primary-hover);
}
```
