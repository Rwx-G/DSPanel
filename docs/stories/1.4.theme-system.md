## Dev Notes

### CSS Architecture

The `src/styles/` directory organizes styles in this order:

```
src/styles/
  tokens.css          # Base design tokens (spacing, typography, elevation)
  theme-light.css     # Light theme color values
  theme-dark.css      # Dark theme color values
  components.css      # Base component styles
  utilities.css       # Utility classes
```

Import order in `src/main.tsx`:
```typescript
import "./styles/tokens.css";
import "./styles/theme-light.css";
import "./styles/theme-dark.css";
import "./styles/components.css";
```

### Runtime Theme Switching

Themes are switched by setting a `data-theme` attribute on the root `<html>` element:

```typescript
// src/hooks/useTheme.ts
import { useState, useEffect, useCallback } from "react";

type ThemeMode = "light" | "dark";

export function useTheme() {
  const [currentTheme, setCurrentTheme] = useState<ThemeMode>(() => {
    const saved = localStorage.getItem("dspanel-theme");
    if (saved === "light" || saved === "dark") return saved;
    return window.matchMedia("(prefers-color-scheme: dark)").matches
      ? "dark"
      : "light";
  });

  const applyTheme = useCallback((mode: ThemeMode) => {
    document.documentElement.setAttribute("data-theme", mode);
    localStorage.setItem("dspanel-theme", mode);
    setCurrentTheme(mode);
  }, []);

  useEffect(() => {
    applyTheme(currentTheme);
  }, []);

  return { currentTheme, applyTheme };
}
```

### Icon System

Using `lucide-react` or inline SVG components:

```typescript
// src/components/Icon.tsx
import { LucideIcon } from "lucide-react";

interface IconProps {
  icon: LucideIcon;
  size?: number;
  className?: string;
}

export function Icon({ icon: IconComponent, size = 16, className }: IconProps) {
  return <IconComponent size={size} className={className} />;
}
```

### Button Component Style Pattern

```css
.btn-primary {
  background-color: var(--color-primary);
  color: var(--color-text-inverse);
  font-size: 14px;
  padding: var(--space-sm) var(--space-lg);
  border: none;
  border-radius: var(--radius-md);
  cursor: pointer;
}

.btn-primary:hover {
  background-color: var(--color-primary-hover);
}
```

### Security Review
No security concerns. Theme preference is stored in localStorage (client-side only).
No sensitive data in theme tokens. The try/catch blocks around localStorage and matchMedia
access prevent information leakage in restricted environments.

### Performance Considerations
Theme switching modifies a single DOM attribute (data-theme on html element), which triggers
CSS custom property recalculation. This is the most performant approach for runtime theme
switching. The useCallback and useState patterns prevent unnecessary re-renders. localStorage
access is synchronous but only happens on theme change and initial load.
