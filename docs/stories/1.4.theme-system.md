# Story 1.4: Theme System, Design Tokens and Resource Architecture

## Status: Draft

## Story

**As a** developer,
**I want** a comprehensive theme system with design tokens, dark/light modes, and organized resource dictionaries,
**so that** the app has a consistent visual identity.

## Acceptance Criteria

1. Color palette is defined as design tokens in `ResourceDictionary` files.
2. Dark and light themes are switchable at runtime without restarting the application.
3. Typography scale defines heading levels 1-4, body, caption, and mono styles.
4. A 4px-based spacing scale provides consistent margins and padding values.
5. Border radius, shadow, and elevation tokens are defined for layered UI depth.
6. Icon system uses vector geometries (`PathGeometry`) for scalable icons.
7. All base WPF control styles are overridden: `Button`, `TextBox`, `ComboBox`, `CheckBox`, `DataGrid`, `TabControl`, `ListBox`, `ScrollBar`.
8. Theme preference is persisted and restored on next launch.
9. Unit tests verify that all expected resources resolve correctly from both themes.

## Dev Notes

### Resource Dictionary Architecture

The `App.xaml` merges dictionaries in this order (later entries override earlier):

```xml
<Application.Resources>
    <ResourceDictionary>
        <ResourceDictionary.MergedDictionaries>
            <!-- Base tokens (spacing, typography, elevation) -->
            <ResourceDictionary Source="Resources/Styles/Typography.xaml"/>

            <!-- Theme (swapped at runtime) -->
            <ResourceDictionary Source="Resources/Styles/LightTheme.xaml"/>

            <!-- Icons -->
            <ResourceDictionary Source="Resources/Icons/IconPaths.xaml"/>

            <!-- Control styles (reference theme colors) -->
            <ResourceDictionary Source="Resources/Styles/Controls.xaml"/>
        </ResourceDictionary.MergedDictionaries>
    </ResourceDictionary>
</Application.Resources>
```

### Runtime Theme Switching

To switch themes at runtime, replace the theme dictionary in `MergedDictionaries`:

```csharp
public interface IThemeService
{
    ThemeMode CurrentTheme { get; }
    void ApplyTheme(ThemeMode mode);
}

public enum ThemeMode
{
    Light,
    Dark
}

public class ThemeService : ObservableObject, IThemeService
{
    private const int ThemeDictionaryIndex = 1; // Index in MergedDictionaries

    [ObservableProperty]
    private ThemeMode _currentTheme;

    public void ApplyTheme(ThemeMode mode)
    {
        var dictionaries = Application.Current.Resources.MergedDictionaries;
        var themeUri = mode switch
        {
            ThemeMode.Dark => new Uri("Resources/Styles/DarkTheme.xaml", UriKind.Relative),
            _ => new Uri("Resources/Styles/LightTheme.xaml", UriKind.Relative)
        };

        dictionaries[ThemeDictionaryIndex] = new ResourceDictionary { Source = themeUri };
        CurrentTheme = mode;
    }
}
```

### Semantic Color Token Names

All color tokens use `DynamicResource` (not `StaticResource`) so they update when the theme changes.

Key naming convention - examples:
- `ColorPrimary`, `ColorPrimaryHover`, `ColorPrimaryPressed`
- `ColorSurfaceBackground`, `ColorSurfaceCard`, `ColorSurfaceElevated`
- `ColorTextPrimary`, `ColorTextSecondary`, `ColorTextDisabled`
- `ColorBorderDefault`, `ColorBorderStrong`
- `ColorSuccess`, `ColorWarning`, `ColorError`, `ColorInfo`

### Spacing Tokens

```xml
<System:Double x:Key="SpaceXs">4</System:Double>
<System:Double x:Key="SpaceSm">8</System:Double>
<System:Double x:Key="SpaceMd">12</System:Double>
<System:Double x:Key="SpaceLg">16</System:Double>
<System:Double x:Key="SpaceXl">24</System:Double>
<System:Double x:Key="Space2xl">32</System:Double>
<System:Double x:Key="Space3xl">48</System:Double>
```

### Icon Geometry Pattern

```xml
<!-- Resources/Icons/IconPaths.xaml -->
<PathGeometry x:Key="IconSearch"
    Figures="M15.5 14h-.79l-.28-.27A6.47 6.47 0 0016 9.5 6.5 6.5 0 109.5 16c1.61 0 3.09-.59 4.23-1.57l.27.28v.79l5 4.99L20.49 19l-4.99-5z"/>
```

Usage in XAML:
```xml
<Path Data="{StaticResource IconSearch}"
      Fill="{DynamicResource ColorTextPrimary}"
      Width="16" Height="16" Stretch="Uniform"/>
```

### Control Style Pattern

All control styles should reference semantic color tokens via `DynamicResource`. Example for Button:

```xml
<Style TargetType="Button" x:Key="PrimaryButton">
    <Setter Property="Background" Value="{DynamicResource ColorPrimary}"/>
    <Setter Property="Foreground" Value="{DynamicResource ColorTextInverse}"/>
    <Setter Property="FontSize" Value="14"/>
    <Setter Property="Padding" Value="16,8"/>
    <Setter Property="BorderThickness" Value="0"/>
    <Setter Property="Cursor" Value="Hand"/>
    <Setter Property="Template">
        <Setter.Value>
            <ControlTemplate TargetType="Button">
                <Border Background="{TemplateBinding Background}"
                        CornerRadius="{StaticResource BorderRadiusMd}"
                        Padding="{TemplateBinding Padding}">
                    <ContentPresenter HorizontalAlignment="Center"
                                      VerticalAlignment="Center"/>
                </Border>
            </ControlTemplate>
        </Setter.Value>
    </Setter>
    <Style.Triggers>
        <Trigger Property="IsMouseOver" Value="True">
            <Setter Property="Background" Value="{DynamicResource ColorPrimaryHover}"/>
        </Trigger>
    </Style.Triggers>
</Style>
```

### Files Created/Modified

| File | Purpose |
|------|---------|
| `Resources/Styles/Colors.xaml` | Base color definitions (if shared across themes) |
| `Resources/Styles/LightTheme.xaml` | Light theme color token values |
| `Resources/Styles/DarkTheme.xaml` | Dark theme color token values |
| `Resources/Styles/Typography.xaml` | Font styles and type scale |
| `Resources/Styles/Controls.xaml` | All control style overrides |
| `Resources/Icons/IconPaths.xaml` | Vector icon geometries |
| `Services/ThemeService.cs` | Runtime theme switching logic |

### Testing

- Testing WPF resource resolution requires creating a `ResourceDictionary` in code and verifying keys exist
- Do not rely on `Application.Current` in unit tests - load dictionaries directly
- Test that both theme dictionaries define identical key sets (no missing keys in dark or light)
- Test `ThemeService` logic by mocking the persistence layer
- Test naming convention: `MethodName_Scenario_ExpectedResult`
