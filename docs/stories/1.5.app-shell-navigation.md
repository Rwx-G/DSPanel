# Story 1.5: Application Shell, Navigation and Tab System

## Status: Done

## Story

**As a** developer,
**I want** a polished app shell with sidebar, tabbed content, breadcrumbs, and status bar,
**so that** all features have consistent navigation.

## Acceptance Criteria

1. Main window has a three-zone layout: sidebar (left), content area (center), status bar (bottom).
2. Sidebar displays grouped module icons with labels and supports collapsing to icon-only mode.
3. Tab system supports opening, closing, closing all, reordering (drag), and overflow scrolling.
4. Breadcrumbs show the current navigation path and allow clicking to navigate back.
5. Status bar displays: connected domain name, domain controller, current permission level, and connection status indicator.
6. Navigation service provides programmatic navigation, tab management, and breadcrumb tracking.
7. Keyboard shortcuts are implemented for common navigation actions.
8. Window state (size, position, maximized) is persisted and restored on next launch.
9. Layout is responsive - sidebar collapses automatically at narrow widths.

### Navigation Context

```typescript
// src/contexts/NavigationContext.tsx
import { createContext, useContext, useState, useCallback } from "react";

interface NavigationState {
    openTabs: TabItem[];
    activeTab: TabItem | null;
    breadcrumbs: BreadcrumbSegment[];
    navigateTo: (moduleId: string, entityId?: string) => void;
    openTab: (title: string, moduleId: string, icon?: string) => TabItem;
    closeTab: (tab: TabItem) => void;
    closeAllTabs: () => void;
    activateTab: (tab: TabItem) => void;
    moveTab: (fromIndex: number, toIndex: number) => void;
}

const NavigationContext = createContext<NavigationState | null>(null);

export function useNavigation() {
    const ctx = useContext(NavigationContext);
    if (!ctx)
        throw new Error("useNavigation must be used within NavigationProvider");
    return ctx;
}
```

### Code Quality Assessment

Code quality is solid overall. Components are well-structured with clear separation of concerns. The NavigationContext centralizes all tab/breadcrumb state with clean `useCallback` usage. The AppShell correctly delegates keyboard shortcuts and responsive sidebar logic. CSS uses design tokens (`--color-*`, `--sidebar-width-*`) consistently, supporting theming. The drag-to-reorder implementation in TabBar is thorough with pointer capture and proper swap calculations.

Minor observations:

- `tabIdCounter` is a module-level mutable variable in NavigationContext.tsx - works but could theoretically cause issues in concurrent test runners. The `resetTabIdCounter()` export mitigates this for tests.
- The `openTab` callback in NavigationContext has `openTabs` in its dependency array, which means a new reference on every tab change. This is a minor performance consideration for downstream consumers.
