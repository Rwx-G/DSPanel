# Story 1.6: Common Reusable Controls

## Status: Draft

## Story

**As a** developer,
**I want** a library of common reusable React components,
**so that** all views use consistent components.

## Acceptance Criteria

1. `SearchBar` component with debounced text input, clear button, placeholder text, and bound search callback.
2. `PermissionGate` component that shows or hides its children based on the user's permission level.
3. `StatusBadge` component displaying a colored pill with text (variants: success, warning, error, info, neutral).
4. `Avatar` component that displays a user image with initials fallback when no image is available.
5. `TagChip` component with text label and optional remove button.
6. `LoadingSpinner` component with animated indicator and optional message text.
7. `EmptyState` component with icon, title, and description for empty data scenarios.
8. `InfoCard` component with header, collapsible content, and optional icon.
9. `CopyButton` component that copies bound text to clipboard and shows visual feedback.
10. All components support both dark and light themes via CSS custom properties.
11. All components support Storybook-style preview or have visual test snapshots.
12. Unit tests cover component logic (debounce, permission checks, clipboard).

## Dev Notes

### SearchBar Component

```typescript
// src/components/common/SearchBar.tsx
import { useState, useEffect, useCallback } from "react";

interface SearchBarProps {
  value: string;
  onChange: (value: string) => void;
  onSearch: (query: string) => void;
  placeholder?: string;
  debounceMs?: number;
}

export function SearchBar({
  value,
  onChange,
  onSearch,
  placeholder = "Search...",
  debounceMs = 300,
}: SearchBarProps) {
  useEffect(() => {
    const timer = setTimeout(() => onSearch(value), debounceMs);
    return () => clearTimeout(timer);
  }, [value, debounceMs, onSearch]);

  return (
    <div className="search-bar">
      <input
        type="text"
        value={value}
        onChange={(e) => onChange(e.target.value)}
        placeholder={placeholder}
      />
      {value && (
        <button onClick={() => { onChange(""); onSearch(""); }}>
          Clear
        </button>
      )}
    </div>
  );
}
```

### PermissionGate Usage

```tsx
<PermissionGate requiredLevel="HelpDesk">
  {/* Content only visible to HelpDesk and above */}
  <button onClick={handleResetPassword}>Reset Password</button>
</PermissionGate>
```

The `PermissionGate` uses the `usePermissions` hook internally to check the current user's permission level.

### StatusBadge Variants

```typescript
type BadgeVariant = "neutral" | "success" | "warning" | "error" | "info";
```

Each variant maps to CSS custom property tokens:
- `neutral` - `var(--color-surface-card)` bg, `var(--color-text-primary)` fg
- `success` - `var(--color-success)` bg (10% opacity), `var(--color-success)` fg
- `warning` - `var(--color-warning)` bg (10% opacity), `var(--color-warning)` fg
- `error` - `var(--color-error)` bg (10% opacity), `var(--color-error)` fg
- `info` - `var(--color-info)` bg (10% opacity), `var(--color-info)` fg

### Avatar Initials Logic

```typescript
export function getInitials(displayName?: string): string {
  if (!displayName?.trim()) return "?";
  const parts = displayName.trim().split(/\s+/);
  if (parts.length === 1) return parts[0][0].toUpperCase();
  return `${parts[0][0]}${parts[parts.length - 1][0]}`.toUpperCase();
}
```

### CopyButton Feedback Flow

1. User clicks button - icon shows "copy" icon
2. `text` is placed on clipboard via `navigator.clipboard.writeText()`
3. Icon swaps to "checkmark" icon
4. After `feedbackMs`, icon reverts to "copy"
5. Use `setTimeout` for the feedback timeout
