# Story 1.6: Common Reusable Controls

## Status: Ready for Review

## Story

**As a** developer,
**I want** a library of common reusable WPF controls,
**so that** all views use consistent components.

## Acceptance Criteria

1. `SearchBar` control with debounced text input, clear button, placeholder text, and bound search command.
2. `PermissionGate` control that shows or hides its content based on the user's permission level.
3. `StatusBadge` control displaying a colored pill with text (variants: success, warning, error, info, neutral).
4. `Avatar` control that displays a user image with initials fallback when no image is available.
5. `TagChip` control with text label and optional remove button.
6. `LoadingSpinner` control with animated indicator and optional message text.
7. `EmptyState` control with icon, title, and description for empty data scenarios.
8. `InfoCard` control with header, collapsible content, and optional icon.
9. `CopyButton` control that copies bound text to clipboard and shows visual feedback.
10. All controls support both dark and light themes via `DynamicResource` bindings.
11. All controls have design-time preview data for XAML designer support.
12. Unit tests cover control logic (debounce, permission checks, clipboard).

## Dev Notes

### SearchBar DependencyProperties

```csharp
namespace DSPanel.Views.Controls;

public partial class SearchBar : UserControl
{
    public static readonly DependencyProperty TextProperty =
        DependencyProperty.Register(nameof(Text), typeof(string), typeof(SearchBar),
            new FrameworkPropertyMetadata(string.Empty,
                FrameworkPropertyMetadataOptions.BindsTwoWayByDefault,
                OnTextChanged));

    public static readonly DependencyProperty SearchCommandProperty =
        DependencyProperty.Register(nameof(SearchCommand), typeof(ICommand), typeof(SearchBar));

    public static readonly DependencyProperty PlaceholderTextProperty =
        DependencyProperty.Register(nameof(PlaceholderText), typeof(string), typeof(SearchBar),
            new PropertyMetadata("Search..."));

    public static readonly DependencyProperty DebounceDelayProperty =
        DependencyProperty.Register(nameof(DebounceDelay), typeof(int), typeof(SearchBar),
            new PropertyMetadata(300));

    private DispatcherTimer? _debounceTimer;

    private static void OnTextChanged(DependencyObject d, DependencyPropertyChangedEventArgs e)
    {
        var searchBar = (SearchBar)d;
        searchBar.RestartDebounceTimer();
    }

    private void RestartDebounceTimer()
    {
        _debounceTimer?.Stop();
        _debounceTimer = new DispatcherTimer
        {
            Interval = TimeSpan.FromMilliseconds(DebounceDelay)
        };
        _debounceTimer.Tick += (s, e) =>
        {
            _debounceTimer.Stop();
            if (SearchCommand?.CanExecute(Text) == true)
                SearchCommand.Execute(Text);
        };
        _debounceTimer.Start();
    }
}
```

### PermissionGate Usage

```xml
<controls:PermissionGate RequiredLevel="HelpDesk">
    <!-- Content only visible to HelpDesk and above -->
    <Button Content="Reset Password" Command="{Binding ResetPasswordCommand}"/>
</controls:PermissionGate>
```

The `PermissionGate` resolves `IPermissionService` from the DI container. One approach is to use a static service locator for controls that cannot use constructor injection:

```csharp
private IPermissionService PermissionService =>
    ((App)Application.Current).Services.GetRequiredService<IPermissionService>();
```

Alternatively, bind `RequiredLevel` and the current level via DataContext, keeping the control DI-free.

### StatusBadge Variants

```csharp
public enum BadgeVariant
{
    Neutral,
    Success,
    Warning,
    Error,
    Info
}
```

Each variant maps to theme color tokens:
- `Neutral` - `ColorSurfaceCard` bg, `ColorTextPrimary` fg
- `Success` - `ColorSuccess` bg (10% opacity), `ColorSuccess` fg
- `Warning` - `ColorWarning` bg (10% opacity), `ColorWarning` fg
- `Error` - `ColorError` bg (10% opacity), `ColorError` fg
- `Info` - `ColorInfo` bg (10% opacity), `ColorInfo` fg

### Avatar Initials Logic

```csharp
public static string GetInitials(string? displayName)
{
    if (string.IsNullOrWhiteSpace(displayName))
        return "?";

    var parts = displayName.Trim().Split(' ', StringSplitOptions.RemoveEmptyEntries);
    return parts.Length switch
    {
        0 => "?",
        1 => parts[0][..1].ToUpperInvariant(),
        _ => $"{parts[0][..1]}{parts[^1][..1]}".ToUpperInvariant()
    };
}
```

### CopyButton Feedback Flow

1. User clicks button - icon shows "copy" icon
2. `CopyText` is placed on clipboard via `Clipboard.SetText()`
3. Icon swaps to "checkmark" icon
4. After `FeedbackDuration` ms, icon reverts to "copy"
5. Use `DispatcherTimer` for the feedback timeout

### Testing

- WPF controls with DependencyProperties cannot be fully unit-tested without a dispatcher
- Extract testable logic (debounce, initials generation, permission checks) into helper classes or static methods
- Test helpers: `Helpers/AvatarHelper.cs` with `GetInitials()`, test in `Helpers/AvatarHelperTests.cs`
- For SearchBar debounce logic, test the timer behavior with a mock command
- For PermissionGate, test the visibility logic with mocked `IPermissionService`
- Tests that require a WPF dispatcher should use `[STAThread]` attribute or a test dispatcher
- Test naming convention: `MethodName_Scenario_ExpectedResult`
