# Story 1.7: Data Display Components

## Status: Done

## Story

**As a** developer,
**I want** standardized data display components for tables, lists, and detail views,
**so that** all data-heavy views look and behave consistently.

## Acceptance Criteria

1. DataTable component supports alternating row colors, hover highlighting, sort indicators on column headers, column resize handles, and frozen columns.
2. FilterBar component displays filter chips, a text filter input, a clear-all button, and emits a filter predicate for data filtering.
3. Pagination component supports configurable page sizes (25/50/100), navigation buttons (first/prev/next/last), and displays total item count.
4. Virtualized list handles 1000+ items with smooth scrolling and no UI lag.
5. PropertyGrid displays label-value pairs grouped by category with copy-on-click support.
6. TreeView is styled with expand/collapse animation, lazy-load children, and multi-select support.
7. DiffViewer supports side-by-side and inline diff modes with color-coded add/remove/unchanged lines.
8. All components support both light and dark themes via CSS custom properties.
9. All components handle empty state, loading state, and error state gracefully.
10. DataTable supports CSV export via a context menu entry.

## Dev Notes

### Architecture

All data display components live in `src/components/data/` as React functional components.

### DataTable Component

DataTable is a custom table component with sorting, resizing, and theming:

```typescript
// src/components/data/DataTable.tsx
interface Column<T> {
  key: keyof T;
  header: string;
  sortable?: boolean;
  frozen?: boolean;
  width?: number;
  render?: (value: T[keyof T], row: T) => ReactNode;
}

interface DataTableProps<T> {
  columns: Column<T>[];
  data: T[];
  onSort?: (key: keyof T, direction: "asc" | "desc") => void;
  onRowClick?: (row: T) => void;
  loading?: boolean;
  emptyMessage?: string;
}
```

### FilterBar

FilterBar exposes filter state via callback. Parent components apply the filter to their data:

```typescript
interface FilterBarProps {
  filters: FilterChip[];
  onFilterChange: (filters: FilterChip[]) => void;
  onTextFilter: (text: string) => void;
}
```

Text input is debounced (300ms) using `setTimeout`.

### VirtualizedList

Uses `@tanstack/react-virtual` for efficient rendering of large lists:

```typescript
import { useVirtualizer } from "@tanstack/react-virtual";

function VirtualizedList<T>({ items, renderItem, itemHeight }: Props<T>) {
  const parentRef = useRef<HTMLDivElement>(null);
  const virtualizer = useVirtualizer({
    count: items.length,
    getScrollElement: () => parentRef.current,
    estimateSize: () => itemHeight,
  });
  // ...
}
```

### DiffViewer

DiffViewer uses two synchronized scroll panels for side-by-side mode. Scroll synchronization is achieved by handling `onScroll` events and setting the other panel's `scrollTop`. The diff algorithm operates on `DiffLine` objects:

```typescript
interface DiffLine {
  type: "unchanged" | "added" | "removed" | "modified";
  content: string;
  oldLineNumber?: number;
  newLineNumber?: number;
}
```

Color mapping is handled via CSS classes with theme-aware custom properties.
