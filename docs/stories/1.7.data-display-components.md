# Story 1.7: Data Display Components

## Status: Draft

## Story

**As a** developer,
**I want** standardized data display components for tables, lists, and detail views,
**so that** all data-heavy views look and behave consistently.

## Acceptance Criteria

1. StyledDataGrid supports alternating row colors, hover highlighting, sort indicators on column headers, column resize handles, and frozen columns.
2. FilterBar control displays filter chips, a text filter input, a clear-all button, and emits a `CollectionViewSource` predicate for data binding.
3. Pagination control supports configurable page sizes (25/50/100), navigation buttons (first/prev/next/last), and displays total item count.
4. VirtualizedList handles 1000+ items with smooth scrolling and no UI lag.
5. PropertyGrid displays label-value pairs grouped by category with copy-on-click support.
6. TreeView is styled with expand/collapse animation, lazy-load children, and multi-select support.
7. DiffViewer supports side-by-side and inline diff modes with color-coded add/remove/unchanged lines.
8. All components support both light and dark themes via resource dictionaries.
9. All components handle empty state, loading state, and error state gracefully.
10. DataGrid supports CSV export via a context menu entry.

## Dev Notes

### Architecture

All data display components live in `src/DSPanel/Views/Controls/` as `UserControl` or custom `Control` subclasses.

### DataGrid Styling

DataGrid styling uses implicit styles defined in `src/DSPanel/Resources/Styles/Controls.xaml`. The implicit style targets `DataGrid` and sets:

- `AlternatingRowBackground` bound to theme resource `DataGridAlternateRowBrush`
- `RowStyle` with `Trigger` on `IsMouseOver` for hover effect
- `ColumnHeaderStyle` with sort direction glyph via `Path` data in a `DataTemplate`
- `GridLinesVisibility="Horizontal"` for clean appearance

### FilterBar

FilterBar exposes a `FilterPredicate` DependencyProperty of type `Predicate<object>`. Parent views bind this to their `CollectionViewSource.Filter`. Internally the control combines text filter and chip filters into a composite predicate. Text input is debounced (300ms) using a `DispatcherTimer`.

### VirtualizedList

Uses the standard WPF `VirtualizingStackPanel` with these settings:

```xml
VirtualizingPanel.VirtualizationMode="Recycling"
VirtualizingPanel.ScrollUnit="Pixel"
VirtualizingPanel.CacheLength="2,2"
VirtualizingPanel.CacheLengthUnit="Page"
```

This ensures smooth pixel-based scrolling and container recycling for large item counts.

### DiffViewer

DiffViewer uses two synchronized `ScrollViewer` controls for side-by-side mode. Scroll synchronization is achieved by handling `ScrollChanged` events and setting the other panel's offset. The diff algorithm operates on `DiffLine` records:

```csharp
public record DiffLine(DiffLineType Type, string Content, int? OldLineNumber, int? NewLineNumber);
public enum DiffLineType { Unchanged, Added, Removed, Modified }
```

Color mapping is handled by `DiffLineTypeToBrushConverter` in `src/DSPanel/Converters/`.
