## Acceptance Criteria

1. FormField component wraps a label, input control, and validation message into a single reusable unit.
2. Styled text input supports normal, focused, error, and disabled visual states with smooth transitions.
3. Styled password input includes a show/hide toggle button for password visibility.
4. Styled select/combobox supports search/filter with a text input inside the dropdown.
5. OUPicker component displays the OU tree from Active Directory and allows single-selection of an OU.
6. GroupPicker component provides a searchable, multi-select list of AD groups.
7. DateTimePicker component allows date and time selection in a single component.
8. Forms use a validation schema (e.g., zod or custom hooks) for field-level and form-level validation.
9. ValidationSummary component aggregates and displays all validation errors from a form.
10. Form dirty tracking detects unsaved changes and can warn before navigation.
11. All components support both light and dark themes.
12. All components are keyboard-navigable and meet basic accessibility requirements (ARIA attributes).

## Dev Notes

### Architecture

Form components reside in `src/components/form/`. Validation logic lives in custom hooks.

### Validation Pattern with Custom Hook

```typescript
// src/hooks/useFormValidation.ts
import { useState, useCallback } from "react";

interface ValidationRule {
  validate: (value: unknown) => boolean;
  message: string;
}

interface FieldConfig {
  rules: ValidationRule[];
}

export function useFormValidation(schema: Record<string, FieldConfig>) {
  const [errors, setErrors] = useState<Record<string, string>>({});

  const validateField = useCallback((name: string, value: unknown) => {
    const config = schema[name];
    if (!config) return;
    for (const rule of config.rules) {
      if (!rule.validate(value)) {
        setErrors((prev) => ({ ...prev, [name]: rule.message }));
        return;
      }
    }
    setErrors((prev) => {
      const next = { ...prev };
      delete next[name];
      return next;
    });
  }, [schema]);

  const validateAll = useCallback((values: Record<string, unknown>) => {
    const newErrors: Record<string, string> = {};
    for (const [name, config] of Object.entries(schema)) {
      for (const rule of config.rules) {
        if (!rule.validate(values[name])) {
          newErrors[name] = rule.message;
          break;
        }
      }
    }
    setErrors(newErrors);
    return Object.keys(newErrors).length === 0;
  }, [schema]);

  return { errors, validateField, validateAll, hasErrors: Object.keys(errors).length > 0 };
}
```

### Custom Validators

Custom validators go in `src/utils/validators.ts`:

```typescript
export function isValidSamAccountName(value: string): boolean {
  return value.length <= 20 && /^[a-zA-Z0-9._\-]+$/.test(value);
}

export function isValidDistinguishedName(value: string): boolean {
  return /^(CN|OU|DC)=.+/.test(value);
}
```

### OUPicker Data Binding

OUPicker calls a Tauri command `get_ou_tree` which returns a tree of `OrganizationalUnit` nodes. Each node has `name`, `distinguished_name`, and `children` (lazy-loaded). The component uses the TreeView from Story 1.7.

### GroupPicker Data Binding

GroupPicker calls a Tauri command `search_groups` with a debounced search string (300ms). Results are displayed in a list with checkbox items. Selected groups are exposed via the `onSelectionChange` callback.

### Dirty Tracking

```typescript
// src/hooks/useChangeTracker.ts
export function useChangeTracker<T extends Record<string, unknown>>(initialValues: T) {
  const [original, setOriginal] = useState(initialValues);
  const [current, setCurrent] = useState(initialValues);

  const isDirty = JSON.stringify(original) !== JSON.stringify(current);

  const markClean = () => setOriginal(current);
  const reset = () => setCurrent(original);

  return { current, setCurrent, isDirty, markClean, reset };
}
```

The navigation guard checks `isDirty` before navigating away and shows a confirmation dialog.
