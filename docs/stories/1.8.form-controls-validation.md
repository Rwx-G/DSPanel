# Story 1.8: Form Controls, Validation and Input Patterns

## Status: Done

## Story

**As a** developer,
**I want** standardized form controls with a validation framework,
**so that** all input forms follow consistent patterns and provide clear feedback.

## Acceptance Criteria

1. FormField control wraps a label, input control, and validation message into a single reusable unit.
2. Styled TextBox supports normal, focused, error, and disabled visual states with smooth transitions.
3. Styled PasswordBox includes a show/hide toggle button for password visibility.
4. Styled ComboBox supports search/filter with a text input inside the dropdown.
5. OUPicker control displays the OU tree from Active Directory and allows single-selection of an OU.
6. GroupPicker control provides a searchable, multi-select list of AD groups.
7. DateTimePicker control allows date and time selection in a single control.
8. ViewModels use `INotifyDataErrorInfo` via CommunityToolkit.Mvvm `ObservableValidator` for validation.
9. ValidationSummary control aggregates and displays all validation errors from a bound ViewModel.
10. Form dirty tracking detects unsaved changes and can warn before navigation.
11. All controls support both light and dark themes.
12. All controls are keyboard-navigable and meet basic accessibility requirements (AutomationProperties).

## Dev Notes

### Architecture

Form controls reside in `src/DSPanel/Views/Controls/`. Validation logic lives in ViewModels using CommunityToolkit.Mvvm's `ObservableValidator`.

### ObservableValidator Pattern

ViewModels that need validation inherit from `ObservableValidator` instead of `ObservableObject`:

```csharp
public partial class EditUserViewModel : ObservableValidator
{
    [ObservableProperty]
    [Required(ErrorMessage = "Display name is required")]
    [MinLength(2, ErrorMessage = "Display name must be at least 2 characters")]
    private string _displayName = string.Empty;

    [RelayCommand]
    private async Task SaveAsync()
    {
        ValidateAllProperties();
        if (HasErrors) return;
        // proceed with save
    }
}
```

The `[ObservableProperty]` source generator automatically calls `ValidateProperty()` on setter, which triggers `INotifyDataErrorInfo.ErrorsChanged`.

### Custom Validation Attributes

Custom attributes go in `src/DSPanel/Validation/`:

```csharp
public class ValidSamAccountNameAttribute : ValidationAttribute
{
    protected override ValidationResult? IsValid(object? value, ValidationContext context)
    {
        if (value is string sam && sam.Length <= 20 && Regex.IsMatch(sam, @"^[a-zA-Z0-9._\-]+$"))
            return ValidationResult.Success;
        return new ValidationResult("SAMAccountName must be 20 chars or less, alphanumeric with . _ -");
    }
}
```

### OUPicker Data Binding

OUPicker calls `IDirectoryProvider.GetOUTreeAsync()` which returns a tree of `OrganizationalUnit` nodes. Each node has `Name`, `DistinguishedName`, and `Children` (lazy-loaded). The control uses a `HierarchicalDataTemplate` for the TreeView.

### GroupPicker Data Binding

GroupPicker calls `IDirectoryProvider.SearchGroupsAsync(query)` with a debounced search string (300ms). Results are displayed in a ListBox with CheckBox items. Selected groups are exposed via `SelectedGroups` DependencyProperty.

### Dirty Tracking

`IChangeTracker` captures a snapshot of property values when `MarkClean()` is called. On each property change, it compares the current value to the snapshot. `IsDirty` returns `true` if any property differs. `NavigationService` checks `IsDirty` before navigating away and shows a confirmation dialog via `IDialogService`.
