# Story 1.9: Dialogs, Notifications and Feedback System

## Status: Done

## Story

**As a** developer,
**I want** a dialog service, toast notification system, and progress feedback,
**so that** user interactions provide clear, consistent feedback.

## Acceptance Criteria

1. `IDialogService` interface exposes `ShowConfirmation`, `ShowWarning`, `ShowError`, `ShowDryRunPreview`, and `ShowCustomDialog<T>` methods.
2. Confirmation dialog displays title, message, an expandable detail section, and configurable buttons (OK/Cancel, Yes/No, custom labels).
3. DryRunPreview dialog displays a scrollable list of pending changes with Cancel and Execute buttons.
4. Toast notifications appear in the bottom-right corner, auto-dismiss after 5 seconds, support severity levels (Info, Success, Warning, Error), can be dismissed by click, and optionally include an action button.
5. Progress dialog supports determinate (percentage) and indeterminate modes, cancellation via CancellationToken, and a status message.
6. Inline progress indicator can be embedded in any view for localized progress feedback.
7. All dialogs and notifications support both light and dark themes.
8. All dialogs support keyboard interaction (Enter to confirm, Escape to cancel).
9. `IDialogService` is registered in DI and fully testable via mock.

## Dev Notes

### Architecture

Dialog infrastructure lives in `src/DSPanel/Services/Dialog/`. Dialog views live in `src/DSPanel/Views/Dialogs/`. The `NotificationHost` control lives in `src/DSPanel/Views/Controls/`.

### IDialogService Implementation

`DialogService` creates WPF `Window` instances for each dialog type. It sets `Owner` to the main window to ensure modality. Return types:

- `ShowConfirmationAsync` -> `Task<bool>` (true = confirmed)
- `ShowWarningAsync` -> `Task` (acknowledge only)
- `ShowErrorAsync` -> `Task` (acknowledge only, optional exception detail)
- `ShowDryRunPreviewAsync` -> `Task<bool>` (true = execute, false = cancel)
- `ShowCustomDialogAsync<TResult>` -> `Task<TResult?>` (result from custom dialog ViewModel)

All methods use `TaskCompletionSource` to bridge WPF dialog result to async/await.

### DryRunChange Model

```csharp
public enum ChangeType { Add, Modify, Delete }

public record DryRunChange(ChangeType Type, string TargetName, string Description);
```

Color mapping: Add = green, Modify = blue, Delete = red.

### Toast Notification System

`NotificationHost` is an `ItemsControl` positioned in a `Canvas` overlay at the bottom-right of `MainWindow.xaml`. Each notification is a `NotificationItem` UserControl with:

- Severity icon and background color
- Message text
- Optional action button
- Close button
- Auto-dismiss via `DispatcherTimer` (5 seconds)
- Slide-in animation on appear, fade-out on dismiss

`NotificationService` manages the `ObservableCollection<NotificationItem>` bound to `NotificationHost`.

### Progress Dialog

```csharp
public record ProgressInfo(double Percentage, string StatusMessage, bool IsIndeterminate = false);
```

`ShowProgressAsync` creates a modal dialog, passes an `IProgress<ProgressInfo>` reporter and a `CancellationToken` to the caller's async work function. The dialog updates its UI via `Progress<ProgressInfo>` which marshals to the UI thread.
