# Story 1.9: Dialogs, Notifications and Feedback System

## Status: Draft

## Story

**As a** developer,
**I want** a dialog service, toast notification system, and progress feedback,
**so that** user interactions provide clear, consistent feedback.

## Acceptance Criteria

1. `useDialog` hook exposes `showConfirmation`, `showWarning`, `showError`, `showDryRunPreview`, and `showCustomDialog` methods.
2. Confirmation dialog displays title, message, an expandable detail section, and configurable buttons (OK/Cancel, Yes/No, custom labels).
3. DryRunPreview dialog displays a scrollable list of pending changes with Cancel and Execute buttons.
4. Toast notifications appear in the bottom-right corner, auto-dismiss after 5 seconds, support severity levels (Info, Success, Warning, Error), can be dismissed by click, and optionally include an action button.
5. Progress dialog supports determinate (percentage) and indeterminate modes, cancellation support, and a status message.
6. Inline progress indicator can be embedded in any view for localized progress feedback.
7. All dialogs and notifications support both light and dark themes.
8. All dialogs support keyboard interaction (Enter to confirm, Escape to cancel).
9. Dialog and notification services are provided via React context and fully testable via mocks.

## Dev Notes

### Architecture

Dialog infrastructure lives in `src/contexts/` and `src/hooks/`. Dialog view components live in `src/components/dialogs/`. The `NotificationHost` component lives in `src/components/common/`.

### Dialog Context Implementation

The dialog system uses React portals and Promise-based resolution:

```typescript
// src/contexts/DialogContext.tsx
import { createContext, useContext, useState, useCallback } from "react";

interface DialogContextValue {
  showConfirmation: (title: string, message: string, detail?: string) => Promise<boolean>;
  showWarning: (title: string, message: string) => Promise<void>;
  showError: (title: string, message: string, detail?: string) => Promise<void>;
  showDryRunPreview: (changes: DryRunChange[]) => Promise<boolean>;
}

export function DialogProvider({ children }: { children: React.ReactNode }) {
  const [dialog, setDialog] = useState<React.ReactNode | null>(null);

  const showConfirmation = useCallback((title: string, message: string, detail?: string) => {
    return new Promise<boolean>((resolve) => {
      setDialog(
        <ConfirmationDialog
          title={title}
          message={message}
          detail={detail}
          onConfirm={() => { setDialog(null); resolve(true); }}
          onCancel={() => { setDialog(null); resolve(false); }}
        />
      );
    });
  }, []);

  // ... other methods

  return (
    <DialogContext.Provider value={{ showConfirmation, showWarning, showError, showDryRunPreview }}>
      {children}
      {dialog}
    </DialogContext.Provider>
  );
}
```

### DryRunChange Type

```typescript
type ChangeType = "add" | "modify" | "delete";

interface DryRunChange {
  type: ChangeType;
  targetName: string;
  description: string;
}
```

Color mapping: add = green, modify = blue, delete = red.

### Toast Notification System

`NotificationHost` is a fixed-position container at the bottom-right of the viewport. Each notification is rendered as a child component with:

- Severity icon and background color
- Message text
- Optional action button
- Close button
- Auto-dismiss via `setTimeout` (5 seconds)
- Slide-in animation on appear, fade-out on dismiss (CSS transitions)

```typescript
// src/hooks/useNotifications.ts
export function useNotifications() {
  const ctx = useContext(NotificationContext);
  return {
    notify: (message: string, severity: NotificationSeverity, action?: NotificationAction) => {
      ctx.addNotification({ message, severity, action });
    },
  };
}
```
