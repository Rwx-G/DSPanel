# Story 10.1: Multi-Format Export (CSV, PDF, XLSX, HTML)

## Status: Draft

## Story

**As a** DSPanel operator,
**I want** to export any list or table view to CSV, PDF, XLSX, or HTML,
**so that** I can share data with stakeholders, auditors, and managers in the format they expect.

## Acceptance Criteria

1. An export button is available on all list and table (React data table) views throughout the application.
2. CSV export produces a UTF-8 file with BOM, with configurable delimiter (comma, semicolon, tab).
3. PDF export includes a title header, generation timestamp, page numbers, and properly formatted table data.
4. XLSX export produces a workbook with a formatted table, auto-filters on columns, and a sheet name matching the view name.
5. HTML export produces a self-contained styled file (inline CSS) suitable for browser viewing or email attachment.
6. A file save dialog opens with a suggested filename based on the view name and current date.
7. A progress indicator is shown for large exports (1000+ rows).
8. Unit tests cover CSV formatting, PDF generation, XLSX output, HTML output, and the export service interface.

## Dev Notes

- `Services/Export/ExportService.rs` - trait interface

- `Models/Export/CsvOptions.rs`
- `Controls/ExportToolbar.tsx`

### Crate Dependencies

- `csv` - CSV generation
- `printpdf` or `genpdf` - PDF generation
- `rust_xlsxwriter` - XLSX generation (no external dependency, pure Rust)
- HTML export uses standard string templating (no extra crate needed)

### Testing

- Use `mockall` (Rust) or vitest mocks to mock `ExportService` trait in React component tests.
- Verify CSV content by reading generated files in tests.
- Verify PDF generation does not throw (content validation with printpdf crate test helpers).
- Verify XLSX output by reading back with a reader or checking file header bytes.
- Verify HTML output is valid markup with expected structure.
- Use standard Rust or TypeScript assertions for readable assertions.
