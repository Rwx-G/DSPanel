# Story 10.2: Scheduled Reports

## Status: Draft

## Story

**As a** DomainAdmin,
**I want** to schedule recurring reports on common AD health indicators,
**so that** reports are generated automatically and available for review without manual effort.

## Acceptance Criteria

1. A scheduler UI allows creating report jobs with: report type, frequency (daily/weekly/monthly), output format (CSV/PDF), and destination folder.
2. Report types include: inactive accounts, expired passwords, passwords never expire, empty groups, orphaned computers, and privileged account summary.
3. Reports execute in the background without blocking the UI.
4. A report history view shows past executions with status (success/failure), timestamp, and output file path.
5. The scheduler is gated to DomainAdmin permission level.
6. Unit tests cover report generation logic, scheduling, and history storage.

## Dev Notes

### Schedule Persistence

Scheduled report definitions are stored in SQLite:

```sql
CREATE TABLE scheduled_reports (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    report_type TEXT NOT NULL,
    frequency TEXT NOT NULL,
    format TEXT NOT NULL,
    destination_path TEXT NOT NULL,
    enabled INTEGER NOT NULL DEFAULT 1,
    last_run TEXT,
    next_run TEXT
);
```
