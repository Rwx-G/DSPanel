# Story 10.4: Compliance Report Templates

## Status: Review

## Story

**As a** DomainAdmin,
**I want** to generate compliance reports from predefined templates (GDPR, HIPAA, SOX, PCI-DSS) with explicit control mapping,
**so that** I can demonstrate AD compliance to auditors with minimal manual preparation.

## Acceptance Criteria

1. A template library view lists available compliance templates with name, standard, and description.
2. Built-in templates are provided for GDPR, HIPAA, SOX, and PCI-DSS standards.
3. Each template defines the data points to collect, presentation format, and sections (summary, findings, evidence, recommendations).
4. Each template section includes a `controlReference` field that maps findings to specific framework controls (e.g., "SOX Section 404", "PCI-DSS Req. 8.1", "GDPR Art. 25").
5. Generated reports display control references alongside findings for audit traceability.
6. Generated reports include a timestamp, generator identity, and report version.
7. Reports are exported as PDF using printpdf crate with professional formatting.
8. Custom templates can be created and edited by the user, with custom control mappings.
9. Unit tests cover template loading, report generation, control mapping display, and PDF output.

## Dev Notes

### Template JSON Schema

```json
{
    "name": "GDPR Access Review",
    "standard": "GDPR",
    "version": "1.0",
    "description": "Review of AD access controls for GDPR compliance",
    "sections": [
        {
            "title": "Privileged Access Summary",
            "controlReference": "GDPR Art. 25 - Data protection by design",
            "type": "query",
            "query": {
                "scope": "privilegedGroups",
                "attributes": [
                    "sAMAccountName",
                    "lastLogonTimestamp",
                    "pwdLastSet"
                ]
            },
            "presentation": "table"
        },
        {
            "title": "Recommendations",
            "controlReference": "GDPR Art. 32 - Security of processing",
            "type": "static",
            "content": "Review all privileged accounts quarterly..."
        }
    ]
}
```

### Built-in Templates

| Template            | Standard | Key Data Points                                        | Control References           |
| ------------------- | -------- | ------------------------------------------------------ | ---------------------------- |
| Access Review       | GDPR     | Privileged accounts, last access, data access groups   | Art. 5, 25, 32               |
| Access Controls     | HIPAA    | Role-based groups, OU structure, audit trail           | 164.312(a), 164.312(b)      |
| Change Management   | SOX      | Admin changes, password policy, account lifecycle      | Section 302, 404             |
| Auth & Access Audit | PCI-DSS  | Authentication settings, access rights, audit logging  | Req. 2, 7, 8, 10            |
