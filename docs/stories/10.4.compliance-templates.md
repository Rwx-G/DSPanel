# Story 10.4: Compliance Report Templates

## Status: Draft

## Story

**As a** DomainAdmin,
**I want** to generate compliance reports from predefined templates (GDPR, HIPAA, SOX),
**so that** I can demonstrate AD compliance to auditors with minimal manual preparation.

## Acceptance Criteria

1. A template library view lists available compliance templates with name, standard, and description.
2. Built-in templates are provided for GDPR, HIPAA, and SOX standards.
3. Each template defines the data points to collect, presentation format, and sections (summary, findings, evidence, recommendations).
4. Generated reports include a timestamp, generator identity, and report version.
5. Reports are exported as PDF using printpdf crate with professional formatting.
6. Custom templates can be created and edited by the user, extending the built-in library.
7. Unit tests cover template loading, report generation, and PDF output.

## Dev Notes

### Template JSON Schema

```json
{
  "name": "GDPR Access Review",
  "standard": "GDPR",
  "version": "1.0",
  "description": "Review of AD access controls for GDPR compliance",
  "sections": [
    {
      "title": "Privileged Access Summary",
      "type": "query",
      "query": {
        "scope": "privilegedGroups",
        "attributes": ["sAMAccountName", "lastLogonTimestamp", "pwdLastSet"]
      },
      "presentation": "table"
    },
    {
      "title": "Recommendations",
      "type": "static",
      "content": "Review all privileged accounts quarterly..."
    }
  ]
}
```

### Built-in Templates

| Template | Standard | Key Data Points |
|----------|----------|----------------|
| Access Review | GDPR | Privileged accounts, last access, data access groups |
| Access Controls | HIPAA | Role-based groups, OU structure, audit trail |
| Change Management | SOX | Admin changes, password policy, account lifecycle |
