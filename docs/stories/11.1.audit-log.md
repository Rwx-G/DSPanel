# Story 11.1: Internal Audit Log

## Status: Draft

## Story

**As a** DomainAdmin,
**I want** a searchable audit log of all write operations performed through DSPanel,
**so that** I can track who did what, when, and investigate issues or policy violations.

## Acceptance Criteria

1. A dedicated audit log view displays a searchable, filterable table with columns: timestamp, user, action, target, details, and result.
2. All write operations across the application (password resets, account modifications, group changes, cleanup actions) feed into the audit log via `AuditService` trait.
3. Audit entries are stored in a local SQLite database.
4. Filters are available for: date range, user, action type, and target DN (partial match).
5. The audit log supports CSV and PDF export.
6. Configurable retention period (default 1 year) with automatic cleanup of older entries.
7. Unit tests cover the audit service, repository, React component filtering, and retention cleanup.

## Dev Notes

### Database Schema

```sql
CREATE TABLE audit_log (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    timestamp TEXT NOT NULL,
    user_name TEXT NOT NULL,
    action_type TEXT NOT NULL,
    target_dn TEXT,
    details TEXT,
    result TEXT NOT NULL
);

CREATE INDEX idx_audit_timestamp ON audit_log(timestamp);
CREATE INDEX idx_audit_action ON audit_log(action_type);
CREATE INDEX idx_audit_user ON audit_log(user_name);
```
