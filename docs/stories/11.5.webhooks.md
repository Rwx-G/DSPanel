# Story 11.5: Webhook Notifications

## Status: Draft

## Story

**As a** DomainAdmin,
**I want** DSPanel to send notifications via webhooks and email when specific events occur,
**so that** my team is alerted in real time through our existing communication channels.

## Acceptance Criteria

1. A configuration UI allows defining notification channels: webhook URL (Teams/Slack compatible) and SMTP email.
2. Subscribable events include: account lockout, privileged group membership change, security alert triggered, and automation rule executed.
3. Notification payloads include event type, timestamp, affected object, operator, and details.
4. A "Test" button sends a test notification to verify channel configuration.
5. A notification history log shows past notifications with status (sent/failed), timestamp, and channel.
6. Unit tests cover webhook sending, email sending, event subscription, and history logging.

## Dev Notes

### Webhook Payload Format (Teams/Slack Compatible)

```json
{
  "text": "DSPanel Alert: Account Lockout",
  "sections": [
    {
      "activityTitle": "Account Lockout Detected",
      "facts": [
        { "name": "Account", "value": "john.doe" },
        { "name": "Time", "value": "2026-03-10T14:30:00Z" },
        { "name": "Source", "value": "DC01.contoso.com" }
      ]
    }
  ]
}
```

### SMTP Configuration

```json
{
  "smtp": {
    "server": "smtp.contoso.com",
    "port": 587,
    "useTls": true,
    "username": "dspanel@contoso.com",
    "from": "dspanel@contoso.com",
    "recipients": ["admin@contoso.com"]
  }
}
```
