# Story 11.6: GPO Viewer

## Status: Draft

## Story

**As a** DomainAdmin,
**I want** to view Group Policy Objects linked to users, computers, and OUs,
**so that** I can understand effective policy application and troubleshoot GPO issues without leaving DSPanel.

## Acceptance Criteria

1. A GPO panel is available on user, computer, and OU detail views showing linked and inherited GPOs.
2. Each GPO entry displays: name, link order, enforcement status, and WMI filter (if any).
3. A "what-if" mode simulates which GPOs would apply to a given user at a specified OU.
4. A scope report for a selected GPO shows all OUs where it is linked.
5. The GPO viewer is read-only (no modifications).
6. The view is gated to DomainAdmin permission level.
7. Unit tests cover GPO link parsing, inheritance resolution, and what-if simulation.

## Dev Notes

### gPLink Attribute Format

The `gPLink` attribute on OU/domain/site objects uses this format:

```
[LDAP://cn={GUID},cn=policies,cn=system,DC=contoso,DC=com;0][LDAP://cn={GUID2},...;2]
```

Link flags:
- `0` = enabled, not enforced
- `1` = disabled
- `2` = enforced
- `3` = disabled and enforced

### GPO Inheritance Rules

1. GPOs are processed from the domain level down to the OU containing the object.
2. Lower OUs override higher OUs (last writer wins) unless the higher GPO is enforced.
3. "Block Inheritance" on an OU stops all non-enforced GPOs from parent OUs.
4. Enforced GPOs always apply regardless of block inheritance.
