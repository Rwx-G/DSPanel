# Story 12.1: Granular RBAC Configuration

## Status: Draft

## Story

**As a** DomainAdmin,
**I want** to configure custom RBAC profiles with feature-level overrides and OU-scoped permissions,
**so that** I can fine-tune access control beyond the four default permission levels.

## Acceptance Criteria

1. A RBAC configuration UI is accessible to DomainAdmin users only.
2. Custom profiles can be created with: name, base permission level, and per-feature overrides (allow/deny).
3. Profiles support OU-scoped permissions (e.g., HelpDesk only for a specific OU subtree).
4. Each profile is mapped to one or more AD security groups.
5. Custom profiles override the default permission detection behavior when configured.
6. Profiles are persisted as JSON files in the preset storage share.
7. All RBAC configuration changes are logged via `AuditService` trait.
8. Unit tests cover profile resolution, OU scoping, and override logic.

## Dev Notes

### Profile JSON Schema

```json
{
    "name": "Paris HelpDesk",
    "baseLevel": "HelpDesk",
    "featureOverrides": {
        "PasswordReset": true,
        "AccountUnlock": true,
        "GroupManagement": false,
        "SecurityDashboard": false
    },
    "scopedOUs": ["OU=Paris,OU=Sites,DC=contoso,DC=com"],
    "mappedGroups": ["CN=Paris-HelpDesk,OU=Groups,DC=contoso,DC=com"]
}
```

### Permission Resolution Order

1. Check if user is member of any custom profile's mapped groups.
2. If matched, use the custom profile's base level + feature overrides.
3. If OU scoping is configured, only apply permissions when target object is within scoped OUs.
4. If no custom profile matches, fall back to default permission detection (well-known group membership).
