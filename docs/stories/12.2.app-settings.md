# Story 12.2: Application Settings

## Status: Done

## Story

**As a** DSPanel user,
**I want** a centralized settings interface organized by category,
**so that** I can configure connection, security, appearance, and other preferences in one place.

## Acceptance Criteria

1. A Settings module is accessible from the sidebar navigation (Settings group).
2. Settings are organized by category tabs: Connection, Presets, Permissions, Security, Reports, Appearance.
3. Connection settings include: domain override, preferred DC, Microsoft Graph configuration (tenant ID, client ID). Graph client secret stored via `CredentialStore` trait (already implemented).
4. Presets settings include: storage path configuration (reuse existing `PresetSettings` component).
5. Permissions tab hosts the permission mapping UI from Story 12.1.
6. Security settings include: audit retention period.
7. Reports settings include: default export format and export directory.
8. Appearance settings include: theme selection (Light, Dark, System).
9. All settings are persisted locally (JSON for non-secrets, OS credential store for secrets) and survive application restarts.
10. Settings are validated before saving (e.g., valid paths, valid URLs, numeric ranges).
11. Unit tests cover settings persistence, validation, defaults, and UI rendering.
