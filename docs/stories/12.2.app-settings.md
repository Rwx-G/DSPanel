# Story 12.2: Application Settings

## Status: Draft

## Story

**As a** DSPanel user,
**I want** a centralized settings interface organized by category,
**so that** I can configure connection, security, appearance, and other preferences in one place.

## Acceptance Criteria

1. A Settings view is accessible from the navigation menu or toolbar.
2. Settings are organized by category tabs: Connection, Presets, Security, Notifications, Reports, Appearance.
3. Connection settings include: domain override, preferred DC, Microsoft Graph configuration (tenant ID, client ID).
4. Presets settings include: storage path configuration (from Story 5.1).
5. Security settings include: MFA configuration and audit retention period.
6. Appearance settings include: theme selection (light/dark) and language selection.
7. All settings are persisted locally (JSON or SQLite) and survive application restarts.
8. Settings are validated before saving (e.g., valid paths, valid URLs, numeric ranges).
9. Unit tests cover settings persistence, validation, and defaults.
