# Story 12.2: Application Settings

## Status: Draft

## Story

**As a** DSPanel user,
**I want** a centralized settings interface organized by category,
**so that** I can configure connection, security, appearance, and other preferences in one place.

## Acceptance Criteria

1. A Settings view is accessible from the navigation menu or toolbar.
2. Settings are organized by category tabs: Connection, Presets, Security, Notifications, Reports, Appearance.
3. Connection settings include: domain override, preferred DC, Microsoft Graph configuration (tenant ID, client ID). Graph client secret is stored in OS credential store via `CredentialStore` trait (already implemented in Epic 6).
4. Presets settings include: storage path configuration (from Story 5.1).
5. Security settings include: MFA configuration and audit retention period.
6. Appearance settings include: theme selection (light/dark) and language selection.
7. All settings are persisted locally (JSON for non-secrets, OS credential store for secrets) and survive application restarts. Secrets must use the `CredentialStore` trait (`services/credential_store.rs`) - never store secrets in JSON.
8. Settings are validated before saving (e.g., valid paths, valid URLs, numeric ranges).
9. Unit tests cover settings persistence, validation, and defaults.

## Dev Notes

### Settings JSON Structure

Non-secret settings in `app-settings.json`:
```json
{
    "connection": {
        "domainOverride": "",
        "preferredDc": "",
        "graphTenantId": "",
        "graphClientId": ""
    },
    "presets": {
        "storagePath": "\\\\server\\share\\DSPanel\\Presets"
    },
    "security": {
        "auditRetentionDays": 365
    },
    "reports": {
        "defaultFormat": "PDF",
        "defaultExportPath": ""
    },
    "appearance": {
        "theme": "System",
        "language": "en"
    }
}
```

Secrets in OS credential store (via `CredentialStore` trait):
- `graph_client_secret` - Azure AD client secret (already implemented)
- `smtp_password` - SMTP credentials (Story 11.5)
- `webhook_slack_url` - Slack webhook URL (Story 11.5)
- `webhook_teams_url` - Teams webhook URL (Story 11.5)

**Rule**: any value that is a password, token, API key, or URL containing auth info
must use `store_credential` / `get_credential` IPC, never `set_app_settings`.
```
