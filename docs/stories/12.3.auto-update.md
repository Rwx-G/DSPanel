# Story 12.3: Auto-Update Notification

## Status: Draft

## Story

**As a** DSPanel user,
**I want** to be notified when a new version of DSPanel is available on GitHub,
**so that** I can stay up to date with the latest features and security fixes.

## Acceptance Criteria

1. At application startup, DSPanel checks the GitHub Releases API for the latest release version.
2. If a newer version is available, a non-blocking notification bar is shown at the top of the main window.
3. The notification includes the new version number and a "Download" button that opens the GitHub release page in the default browser.
4. "Skip This Version" dismisses the notification permanently for that version.
5. "Remind Me Later" dismisses the notification until the next startup.
6. The check frequency is configurable in Settings (every startup, daily, weekly, never).
7. When offline or if the API is unreachable, the check fails silently without affecting the application.
8. Unit tests cover version comparison, notification logic, skip/remind behavior, and frequency check.

## Tasks / Subtasks

- [ ] Task 1: UpdateInfo model and service trait (AC: #1)
    - [ ] Create `UpdateInfo` struct: `version`, `release_url`, `release_notes`, `published_at`
    - [ ] Create `UpdateCheckResult`: `Available(UpdateInfo)` | `UpToDate` | `Skipped` | `CheckDisabled`
    - [ ] Add update settings to `AppSettings`: `skipped_version`, `last_update_check`, `update_check_frequency`
- [ ] Task 2: GitHub Releases API integration (AC: #1, #7)
    - [ ] Use existing `http_client` from `AppState` to GET `https://api.github.com/repos/Rwx-G/DSPanel/releases/latest`
    - [ ] Parse JSON response: `tag_name`, `html_url`, `body`, `published_at`
    - [ ] Handle all errors silently (timeout, DNS, HTTP errors) - return `None`
    - [ ] Set User-Agent header (already configured in http_client)
- [ ] Task 3: Semantic version comparison (AC: #1)
    - [ ] Parse version strings: strip leading 'v', split on '.', compare major.minor.patch
    - [ ] Handle edge cases: pre-release tags (ignore them), malformed versions
    - [ ] Compare against `env!("CARGO_PKG_VERSION")` for current version
- [ ] Task 4: Frequency and skip logic (AC: #4, #5, #6)
    - [ ] Check frequency: compare `last_update_check` with current time based on setting
    - [ ] Skip logic: if latest version == `skipped_version`, return `Skipped`
    - [ ] After successful check, update `last_update_check` in settings
- [ ] Task 5: Tauri IPC commands (AC: #1, #4, #5, #6)
    - [ ] `check_for_update() -> Option<UpdateInfo>` - runs the full check pipeline
    - [ ] `skip_update_version(version: String)` - persists skipped version
    - [ ] `open_release_page(url: String)` - opens URL in default browser via `tauri::api::shell::open`
- [ ] Task 6: Update notification bar UI (AC: #2, #3, #4, #5)
    - [ ] `UpdateNotificationBar.tsx` - top bar in AppShell, above content area
    - [ ] Shows: "DSPanel vX.Y.Z is available" with info icon
    - [ ] Buttons: "Download" (opens release page), "Skip This Version", "Remind Me Later"
    - [ ] Smooth slide-in animation, non-blocking
    - [ ] Uses existing design tokens for consistent styling
- [ ] Task 7: Integration in AppShell (AC: #2)
    - [ ] Call `check_for_update()` on AppShell mount (after initial load)
    - [ ] Conditionally render `UpdateNotificationBar` above content
    - [ ] Dismiss bar on any button click
- [ ] Task 8: Settings integration (AC: #6)
    - [ ] Add "Updates" section in Connection or Appearance tab of Settings (Story 12.2)
    - [ ] Dropdown: check frequency (Every Startup, Daily, Weekly, Never)
- [ ] Task 9: Write tests (AC: #8)
    - [ ] Rust: test version comparison (newer, same, older, pre-release, malformed)
    - [ ] Rust: test skip version logic
    - [ ] Rust: test frequency check respects interval
    - [ ] Rust: test API error returns None
    - [ ] Frontend: test UpdateNotificationBar renders with version info
    - [ ] Frontend: test Download button calls open_release_page
    - [ ] Frontend: test Skip calls skip_update_version
    - [ ] Frontend: test Remind Later just dismisses

## Dev Notes

### GitHub API Response

```json
{
  "tag_name": "v1.2.0",
  "html_url": "https://github.com/Rwx-G/DSPanel/releases/tag/v1.2.0",
  "body": "## What's New\n- Feature X\n- Fix Y",
  "published_at": "2026-03-20T10:00:00Z"
}
```

### Update Settings in AppSettings

```json
{
  "update": {
    "checkFrequency": "startup",
    "skippedVersion": null,
    "lastCheckTimestamp": "2026-03-25T08:00:00Z"
  }
}
```
