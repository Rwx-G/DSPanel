# Story 12.3: Auto-Update Notification

## Status: Draft

## Story

**As a** DSPanel user,
**I want** to be notified when a new version of DSPanel is available on GitHub,
**so that** I can stay up to date with the latest features and security fixes.

## Acceptance Criteria

1. At application startup, DSPanel checks the GitHub Releases API for the latest release version.
2. If a newer version is available, a non-blocking notification bar is shown at the top of the main window.
3. The notification includes the new version number and a "Download" button that opens the GitHub release page in the default browser.
4. "Skip This Version" dismisses the notification permanently for that version.
5. "Remind Me Later" dismisses the notification until the next startup.
6. The check frequency is configurable (daily, weekly, never).
7. When offline or if the API is unreachable, the check fails silently without affecting the application.
8. Unit tests cover version comparison, notification logic, and skip/remind behavior.

## Dev Notes

### GitHub API Request

```typescript
var client = new reqwest client();
client.DefaultRequestHeaders.UserAgent.ParseAdd("DSPanel/1.0");
var response = await client.GetAsync(
    "https://api.github.com/repos/{owner}/{repo}/releases/latest");
```

The API returns JSON with `tag_name` (e.g., "v1.2.0"), `html_url` (release page), and `body` (release notes markdown).

### Version Comparison

Use `semver crate` or a custom semantic version parser. Handle formats like `v1.2.0`, `1.2.0`, `v1.2.0-beta.1`.
