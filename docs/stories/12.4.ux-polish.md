# Story 12.4: UX Polish and Final Touches

## Status: Draft

## Story

**As a** DSPanel user,
**I want** a polished, consistent, and accessible user experience across all views,
**so that** the application feels professional and is efficient to use daily.

## Acceptance Criteria

1. Dark and light themes have proper contrast ratios in all views (WCAG AA minimum).
2. Keyboard shortcuts are defined for common actions: Ctrl+F (search), Ctrl+R (refresh), Ctrl+E (export), Escape (close dialog).
3. All async operations show a loading indicator (spinner or progress bar).
4. Error messages are user-friendly with actionable guidance (not raw exception text).
5. Window position, size, and state (maximized/normal) are persisted between sessions.
6. The application has a custom icon and branding in the title bar and taskbar.
7. An About dialog shows version, license, author, and links to documentation and GitHub.
8. Unit tests cover window state persistence, keyboard shortcut routing, and About dialog.

### Custom Event Pattern for View Actions

```typescript
// In AppShell.tsx keyboard handler:
window.dispatchEvent(new CustomEvent("dspanel:refresh"));

// In a page component:
useEffect(() => {
  const handler = () => refreshData();
  window.addEventListener("dspanel:refresh", handler);
  return () => window.removeEventListener("dspanel:refresh", handler);
}, []);
```
