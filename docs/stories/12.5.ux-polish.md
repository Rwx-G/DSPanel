# Story 12.5: UX Polish and Final Touches

## Status: Draft

## Story

**As a** DSPanel user,
**I want** a polished, consistent, and accessible user experience across all views,
**so that** the application feels professional and is efficient to use daily.

## Acceptance Criteria

1. Dark and light themes have proper contrast ratios in all views (WCAG AA minimum).
2. Keyboard shortcuts are defined for common actions: Ctrl+F (search), Ctrl+R (refresh), Ctrl+E (export), Escape (close dialog).
3. All async operations show a loading indicator (spinner or progress bar).
4. Error messages are user-friendly with actionable guidance (not raw exception text).
5. Window position, size, and state (maximized/normal) are persisted between sessions.
6. The application has a custom icon and branding in the title bar and taskbar.
7. An About dialog shows version, license, author, and links to documentation and GitHub.
8. Unit tests cover window state persistence and keyboard shortcut routing.

## Dev Notes

### Window State Persistence

```csharp
public class WindowState
{
    public double Left { get; set; }
    public double Top { get; set; }
    public double Width { get; set; }
    public double Height { get; set; }
    public System.Windows.WindowState State { get; set; }
}
```

Save in `settings.json` under `"windowState"`. On restore, verify the saved position is within the bounds of current monitors using `System.Windows.Forms.Screen.AllScreens` or `System.Windows.SystemParameters`.

### Keyboard Shortcuts

| Shortcut | Action | Scope |
|----------|--------|-------|
| Ctrl+F | Focus search / open search | Global |
| Ctrl+R | Refresh current view | Global |
| Ctrl+E | Export current view | Views with export |
| Escape | Close dialog / clear search | Global |
| F5 | Refresh current view | Global |
| Ctrl+S | Save (in editors) | Editor views |

### Error Message Pattern

```csharp
try { /* operation */ }
catch (LdapException ex) when (ex.ErrorCode == 49)
{
    logger.Error(ex, "LDAP bind failed for {User}", user);
    dialogService.ShowError(
        Resources.Error_InsufficientPermissions,
        Resources.Error_InsufficientPermissions_Detail);
}
```
