# Story 13.1: i18n Infrastructure Setup

## Status: Review

## Story

**As a** developer,
**I want** a localization framework integrated into DSPanel,
**so that** all user-facing strings can be translated without code changes.

## Acceptance Criteria

1. i18next and react-i18next are installed and configured.
2. All user-facing strings are externalized to JSON translation files.
3. English (en) is the default language with complete coverage.
4. Translation file structure is organized by feature/view for maintainability.
5. Language is hot-switchable at runtime (no restart required).
6. Date, number, and currency formatting follows the selected locale.
7. Language selection is available in Settings > Appearance tab.
8. Developer documentation explains how to add new languages.
9. Unit tests verify all resource keys exist and locale switching works.

## Dev Notes

### Translation File Structure

```json
{
  "common": {
    "save": "Save",
    "cancel": "Cancel",
    "delete": "Delete",
    "search": "Search",
    "loading": "Loading...",
    "error": "An error occurred",
    "noResults": "No results found"
  },
  "sidebar": {
    "directory": "Directory",
    "infrastructure": "Infrastructure",
    "security": "Security"
  },
  "userLookup": {
    "title": "User Lookup",
    "searchPlaceholder": "Search by name, SAM, or email..."
  }
}
```

### i18n Initialization

```typescript
// src/i18n.ts
import i18n from "i18next";
import { initReactI18next } from "react-i18next";
import en from "./locales/en.json";

i18n.use(initReactI18next).init({
  resources: { en: { translation: en } },
  lng: "en",
  fallbackLng: "en",
  interpolation: { escapeValue: false },
});

export { i18n };
```

### Key Naming Convention

- `{view}.{element}` pattern: `userLookup.title`, `passwordReset.successMessage`
- `common.*` for shared strings: `common.save`, `common.cancel`
- `sidebar.*` for navigation labels
- `errors.*` for error messages
