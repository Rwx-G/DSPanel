# Story 13.2: Translations

## Status: Review

## Story

**As a** DSPanel user,
**I want** the application to be available in my language,
**so that** I can use the tool in my preferred language.

## Acceptance Criteria

1. Complete translation files are provided for all 5 languages: English (en), French (fr), German (de), Italian (it), Spanish (es).
2. All views, dialogs, notifications, and error messages are translated.
3. AD-specific technical terms are kept in English where appropriate (DN, OU, GPO, SID, LDAP, etc.).
4. Pluralization rules are handled correctly for each language.
5. Date formatting follows each locale convention (fr: dd/MM/yyyy, de: dd.MM.yyyy, it: dd/MM/yyyy, es: dd/MM/yyyy).
6. Unit tests verify all translation keys exist in all language files with no missing entries.

## Dev Notes

### Terms to Keep in English

These AD/IT terms should NOT be translated:
- DN, OU, GPO, SID, LDAP, NTFS, MFA, HIBP
- Domain Controller (DC), Active Directory (AD)
- SAM Account Name, UPN, Distinguished Name
- CSV, PDF, JSON
- Exchange, Entra ID, Graph API

### Pluralization Example

```json
{
  "userCount": "{{count}} user",
  "userCount_plural": "{{count}} users"
}
```

i18next handles pluralization rules per language automatically.

### File Layout

- `src/locales/fr.json` - French translations
- `src/locales/de.json` - German translations
- `src/locales/it.json` - Italian translations
- `src/locales/es.json` - Spanish translations
- `src/i18n.ts` - register all languages

### Testing

- Frontend: vitest tests for translation completeness
- Test file: `src/locales/translations.test.ts`
