# Story 2.3: Account Unlock and Enable/Disable

## Status: Done

## Story

**As a** HelpDesk operator,
**I want** to unlock, enable, or disable a user account directly from the lookup view,
**so that** I can resolve access issues quickly without switching to external AD tools.

## Acceptance Criteria

1. The user detail view shows "Unlock", "Enable", and "Disable" action buttons, each gated to HelpDesk+ permission level.
2. "Unlock" is only visible/enabled when the account is currently locked out.
3. "Enable" is only visible/enabled when the account is currently disabled.
4. "Disable" is only visible/enabled when the account is currently enabled.
5. Each action triggers a confirmation dialog via `DialogService` trait before execution.
6. On success, the user detail view refreshes immediately to reflect the new account state.
7. On failure, a clear error message is displayed to the operator.
8. Every action - success or failure - is logged via `AuditService` trait.
9. Unit tests cover React component logic, button visibility states, confirmation flow, and audit logging.

## Dev Notes

### UserAccountControl Flags

Account state is managed via the `userAccountControl` LDAP attribute using bitwise flags:
- `ACCOUNTDISABLE = 0x0002` - set to disable, clear to enable
- Lockout is managed via the `lockoutTime` attribute - set to `0` to unlock

### DirectoryProvider trait Methods

```typescript
Task UnlockAccountAsync(string userDn);    // Sets lockoutTime = 0
Task EnableAccountAsync(string userDn);     // Clears ACCOUNTDISABLE flag
Task DisableAccountAsync(string userDn);    // Sets ACCOUNTDISABLE flag
```

### Button State Logic

The React component exposes computed properties that drive button visibility:
- `CanUnlock` - true when `IsLockedOut == true`
- `CanEnable` - true when `IsDisabled == true`
- `CanDisable` - true when `IsDisabled == false`

These properties update after each refresh via React state updates.

### Confirmation Dialog

`DialogService trait.ConfirmAsync(title, message)` returns `bool`. The message includes the target user's display name and a clear description of the action, e.g., "Are you sure you want to disable the account for John Smith (jsmith)?".
