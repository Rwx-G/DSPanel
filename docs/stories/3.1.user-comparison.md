# Story 3.1: Side-by-Side User Comparison

## Status: Draft

## Story

**As a** HelpDesk operator,
**I want** to compare two AD users side by side with a group membership delta,
**so that** I can quickly identify permission differences and troubleshoot access issues.

## Acceptance Criteria

1. A "Compare" tab/page allows selecting two users via search fields.
2. Both users' key attributes are displayed in parallel columns (display name, title, department, OU, last logon, account status).
3. Group memberships are compared with color-coded indicators: green for shared groups, red for groups only in User A, blue for groups only in User B.
4. A delta summary shows counts: total shared, only-A, only-B.
5. Group lists support filtering by name and sorting by name or membership type (shared/unique).
6. The comparison view opens as a new tab in the main navigation.
7. Unit tests cover the comparison logic, delta computation, and React component bindings.

## Dev Notes

### Group Diff Algorithm

The comparison uses set operations on group distinguished names (case-insensitive):
- `SharedGroups = groupsA.Intersect(groupsB, StringComparer.OrdinalIgnoreCase)`
- `OnlyAGroups = groupsA.Except(groupsB, StringComparer.OrdinalIgnoreCase)`
- `OnlyBGroups = groupsB.Except(groupsA, StringComparer.OrdinalIgnoreCase)`

### Group Resolution

Use `DirectoryProvider trait.GetUserGroupsAsync(userDn)` which should return all effective group memberships. This can leverage the `tokenGroups` constructed attribute for recursive resolution or perform manual recursive expansion of `memberOf`.
