# Story 3.4: State-in-Time Comparison

## Status: Draft

## Story

**As an** IT administrator,
**I want** to view replication metadata timestamps for AD object attributes and compare states at different points in time,
**so that** I can investigate when specific changes occurred and understand the history of an object.

## Acceptance Criteria

1. The detail view for users, computers, and groups shows a "History" tab with attribute change timestamps from replication metadata.
2. Replication metadata is queried from the `msDS-ReplAttributeMetaData` operational attribute.
3. A timeline view shows when each attribute was last modified, sorted by date.
4. The operator can select two timestamps and see a before/after diff for the attributes that changed between them.
5. The diff view highlights added, removed, and changed values.
6. The feature degrades gracefully when metadata is limited or unavailable (e.g., attributes not yet replicated, missing metadata).
7. Unit tests cover metadata parsing, timeline construction, and diff computation.

## Dev Notes

### msDS-ReplAttributeMetaData Format

The `msDS-ReplAttributeMetaData` operational attribute returns an XML fragment per attribute:

```xml
<DS_REPL_ATTR_META_DATA>
  <pszAttributeName>displayName</pszAttributeName>
  <dwVersion>3</dwVersion>
  <ftimeLastOriginatingChange>2026-02-15T14:30:00Z</ftimeLastOriginatingChange>
  <uuidLastOriginatingDsaInvocationID>...</uuidLastOriginatingDsaInvocationID>
  <usnOriginatingChange>12345</usnOriginatingChange>
  <usnLocalChange>67890</usnLocalChange>
</DS_REPL_ATTR_META_DATA>
```

### Querying Operational Attributes

Operational attributes are not returned by default. The LDAP search request must explicitly list `msDS-ReplAttributeMetaData` in the requested attributes array. Use `ldap3 crate.LDAP search request` with this attribute name.

### Diff Limitations

The replication metadata only tells us _when_ each attribute was last changed and how many times (version count). It does not store previous values. The "diff" between two timestamps shows which attributes changed during that interval (based on version increments and timestamps), but the actual before-value is not available from metadata alone.

To work around this: display current values alongside change metadata, and clearly indicate that historical values are not available - only change timestamps and version counts.

### Linked Attributes

Linked attributes (e.g., `member`, `memberOf`) may not have replication metadata in the same format. These attributes use linked-value replication with `msDS-ReplValueMetaData` instead. Consider supporting this as a future enhancement.

### AttributeMetadata Model

```typescript
pub struct AttributeMetadata(
    string AttributeName,
    int Version,
    DateTime LastOriginatingChangeTime,
    string LastOriginatingDsaDn,
    long LocalUsn,
    long OriginatingUsn);
```
