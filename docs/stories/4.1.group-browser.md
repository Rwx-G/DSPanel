# Story 4.1: Group Browser (Tree and Flat View)

## Status: Draft

## Story

**As an** IT administrator,
**I want** to browse AD groups in both a hierarchical OU tree and a flat searchable list,
**so that** I can quickly find and inspect any group in the domain.

## Acceptance Criteria

1. A "Group Management" page is available in the main navigation.
2. A tree view displays groups organized by Organizational Unit hierarchy, with lazy loading on OU expansion.
3. A flat view displays all groups in a searchable, sortable list.
4. A toggle allows switching between tree and flat views.
5. Selecting a group displays its details: name, distinguished name, description, scope (Global/DomainLocal/Universal), category (Security/Distribution), and member count.
6. The detail panel shows a member list with type icons (user, group, computer) for each member.
7. Search works across both views - filtering the tree or the flat list by group name.
8. Unit tests cover tree construction, flat list filtering, detail loading, and member display.

## Dev Notes

### GroupType Attribute Parsing

The `groupType` attribute is a bitwise integer:

- Scope bits: `GLOBAL = 0x2`, `DOMAIN_LOCAL = 0x4`, `UNIVERSAL = 0x8`
- Category bit: `SECURITY = 0x80000000` (if set, Security group; if not, Distribution group)

Example: `groupType = -2147483646` (0x80000002) means Global Security group.

### Lazy Loading Pattern

```typescript
    public bool IsExpanded { get; set; } // triggers lazy load
    private bool _isLoaded;

    // On first expand, replace dummy child with actual children

```

### LDAP Queries

- Root OUs: `(&(objectClass=organizationalUnit))` scoped to one level below domain root
- Child OUs: same filter, scoped to one level below selected OU
- Groups in OU: `(objectClass=group)` scoped to one level below OU
- All groups (flat): `(objectClass=group)` with paged results control

### Cross-module deep-link from UserLookup

The UserLookup page has a "View group members" dialog (right-click on a group
membership row). This dialog should include a button to open the Group
Management module with that group pre-selected, providing a seamless
drill-down from user detail into full group management. The navigation
mechanism (e.g. `navigation.openTab("groups", { selectedGroupDn })`) and
the GroupManagement page's ability to accept and select an incoming group DN
should be implemented as part of this story.

### Performance Considerations

Large domains can have thousands of groups. The flat view should use server-side paging (LDAP paged results control) and load incrementally. The tree view naturally limits queries via lazy loading.
