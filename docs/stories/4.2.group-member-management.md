# Story 4.2: Group Member Management

## Status: Draft

## Story

**As an** Account Operator,
**I want** to add and remove group members using drag-drop and multi-select with a dry-run preview,
**so that** I can manage group memberships efficiently while reviewing changes before they take effect.

## Acceptance Criteria

1. Drag-and-drop from user search results or other group member lists adds members to the selected group.
2. Multi-select allows batch adding of multiple users/groups at once.
3. Members can be removed via delete key or right-click context menu.
4. All add/remove operations show a dry-run preview before execution.
5. Every membership change is logged via `AuditService` trait with operator, group, and member details.
6. Operations are gated to AccountOperator+ permission level.
7. A confirmation summary shows total adds and removes before applying.
8. Unit tests cover drag-drop logic, batch operations, dry-run preview, permission gating, and audit logging.

## Dev Notes

### Drag-Drop Implementation

React or Tauri drag-drop uses `React DnD.DoReact DnD()` on the source and `Drop` event handler on the target. The data object should carry a `List<string>` of member DNs. Use `React DnD data transfer` with a custom format like `DSPanel.MemberDnList`.

### Batch Operation Flow

1. Operator drags users or clicks "Add Selected" - pending changes collected
2. Operator optionally removes members - pending changes collected
3. Operator clicks "Apply" - dry-run preview dialog opens
4. Preview shows all pending adds/removes
5. Operator confirms - changes applied sequentially
6. Each change is individually audited
7. On partial failure, completed changes are kept and errors are reported

### LDAP Member Modification

Adding a member to a group modifies the `member` attribute of the group object:

- Add: LDAP modify request with `LDAP attribute modification` of type `Add`
- Remove: LDAP modify request with `LDAP attribute modification` of type `Delete`

### Circular Nesting Prevention

Before adding a group as a member of another group, check if the target group is already a member (direct or nested) of the source group. This prevents circular nesting like A contains B contains A.

**Design Divergences (approved during implementation):**
- AC #1: Drag-and-drop replaced by search dropdown. DnD adds complexity (React DnD library, drag handles, drop zones) for marginal UX benefit in a desktop admin tool. The dropdown pattern is consistent with Azure AD/Entra portal UX.
- AC #2: Batch add via dropdown is per-item (click to stage). The staging + preview workflow allows accumulating multiple adds before applying.
- AC #3: Delete key and context menu remove replaced by checkbox + "Remove" button. The unified member list with checkboxes + always-visible "Remove" button is more discoverable.
