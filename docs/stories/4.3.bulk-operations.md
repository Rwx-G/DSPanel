# Story 4.3: Bulk Operations (Delete/Add/Transfer)

## Status: Done

## Story

**As an** Account Operator,
**I want** to perform bulk group membership operations (delete, add, transfer) with dry-run preview and rollback,
**so that** I can efficiently restructure group memberships for large sets of users with confidence.

## Acceptance Criteria

1. A bulk operations panel allows selecting source group(s), target group(s), and an operation type: Delete, Add, or Transfer.
2. The operator can select specific members from the source group(s) or apply the operation to all members.
3. Transfer is implemented as Add to target + Delete from source, executed atomically.
4. A dry-run preview shows every individual change that will be made before execution.
5. A progress indicator shows real-time status during execution (X of Y completed).
6. On partial failure, completed changes are rolled back and the operator is notified with details.
7. Every operation - success, failure, and rollback - is logged via `AuditService` trait.
8. Unit tests cover all three operation types, dry-run, progress tracking, rollback, and audit logging.

## Dev Notes

### Operation Models

```typescript
pub enum BulkOperationType { Delete, Add, Transfer }

pub struct BulkOperation(
    BulkOperationType Type,
    IReadOnlyList<string> SourceGroupDns,
    IReadOnlyList<string> TargetGroupDns,
    IReadOnlyList<string> MemberDns);

pub struct PlannedChange(
    string MemberDn,
    string GroupDn,
    ChangeAction Action);  // Add or Remove

pub enum ChangeAction { Add, Remove }
```

### Transfer Atomicity

Transfer for a single member is a two-step operation:

1. Add member to target group
2. Remove member from source group

If step 1 fails, skip step 2 and report error. If step 2 fails, rollback step 1 (remove from target). This ensures the member is never lost from both groups.

### Rollback Strategy

The rollback log stores each completed change as a `PlannedChange`. On failure, iterate the log in reverse and apply the inverse operation:

- For a completed `Add`, apply `Remove`
- For a completed `Remove`, apply `Add`

If a rollback operation itself fails, log it but continue with remaining rollback items. Report all failures to the operator.

### Progress Reporting

Use `Progress<BulkOperationProgress>` to report progress from the service layer to the React component. The React component updates `ProgressCurrent`, `ProgressTotal`, and `StatusMessage` React state properties.

### Performance

For large batch operations (hundreds of members), consider using LDAP batch modify requests. However, the rollback mechanism requires tracking individual operations, so each add/remove is issued as a separate LDAP request.
