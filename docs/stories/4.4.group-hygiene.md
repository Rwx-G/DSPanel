# Story 4.4: Empty and Circular Group Detection

## Status: Draft

## Story

**As an** IT administrator,
**I want** to detect empty groups and circular group nesting in my domain,
**so that** I can clean up unused groups and fix nesting issues that cause unexpected behavior.

## Acceptance Criteria

1. A "Hygiene" tab in the Group Management page lists all empty groups in the domain.
2. Circular group nesting is detected and reported with the full cycle path (e.g., A -> B -> C -> A).
3. One-click navigation from a hygiene finding jumps to the problematic group in the group browser.
4. Empty groups can be deleted in bulk with a dry-run preview before execution.
5. Bulk delete is gated to AccountOperator+ permission level.
6. Every delete operation is logged via `AuditService` trait.
7. Unit tests cover empty group detection, cycle detection, navigation, bulk delete, and audit logging.

## Dev Notes

### Cycle Detection Algorithm

Use a DFS-based algorithm with three colors (white/gray/black):

1. Start: all nodes white (unvisited)
2. On visit: mark gray (in current path)
3. If a neighbor is gray, a cycle is found - trace back to build cycle path
4. On finish: mark black (fully explored)

### Empty Group Filtering

Query with `(&(objectClass=group)(!(member=*)))` to find groups with no `member` attribute. However, this filter may not work on all AD implementations - an alternative is to query all groups and check `member` count client-side.

Built-in groups to exclude: groups in `CN=Builtin,DC=...` and well-known groups like Domain Users, Domain Computers, Domain Controllers.

### GroupCycle Model

```typescript
pub struct GroupCycle(IReadOnlyList<GroupCycleNode> Path);

pub struct GroupCycleNode(string GroupDn, string GroupName);
```

### Bulk Delete

Group deletion uses `DirectoryProvider trait.DeleteObjectAsync(string dn)`. Before deletion, verify the group is still empty (re-check to avoid race conditions). Deletion of non-empty groups should be blocked.

### Performance

For large domains, the hygiene scan can be expensive. Run it on demand (button click) rather than on page load. Show a progress indicator with estimated completion. The membership graph is built from a single LDAP query that retrieves all groups with their `member` attributes.
