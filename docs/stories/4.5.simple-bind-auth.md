# Story 4.5: Simple Bind Authentication for LDAP

## Status: Done

## Story

**As a** developer,
**I want** DSPanel to support LDAP simple bind (username/password) as an alternative to GSSAPI (Kerberos),
**so that** I can test against a standalone AD lab without joining my dev machine to the test domain.

## Acceptance Criteria

1. `LdapDirectoryProvider` supports simple bind authentication using a bind DN and password.
2. Authentication mode is selected at runtime based on configuration (not compile-time feature flags).
3. When no credentials are configured, GSSAPI remains the default (existing behavior unchanged).
4. Simple bind credentials are read from environment variables: `DSPANEL_LDAP_BIND_DN`, `DSPANEL_LDAP_BIND_PASSWORD`, and `DSPANEL_LDAP_SERVER`.
5. The `DSPANEL_LDAP_SERVER` variable overrides the auto-detected domain for the LDAP connection URL.
6. Simple bind mode logs a warning at startup indicating credentials-based auth is in use.
7. Credentials are never logged, even at trace level.
8. Unit tests cover both auth paths: GSSAPI fallback and simple bind selection.
9. Integration test scaffold exists in `src-tauri/tests/` for manual testing against a real AD.

## Dev Notes

### Auth Mode Enum

```rust
pub enum LdapAuthMode {
    /// Kerberos authentication via GSSAPI (default, requires domain-joined machine)
    Gssapi,
    /// Simple bind with explicit credentials (for lab/test environments)
    SimpleBind {
        bind_dn: String,
        password: String,
    },
}
```

### Connection URL Override

When `DSPANEL_LDAP_SERVER` is set, use it directly as the LDAP host instead of the `USERDNSDOMAIN` env var. This allows connecting to a DC by IP (e.g., `10.0.0.10`) when the dev machine doesn't resolve the test domain DNS.

```rust
let host = match &self.server_override {
    Some(server) => server.clone(),
    None => self.domain.as_ref()
        .context("No domain available")?.clone(),
};
let url = format!("ldap://{}:389", host);
```

### Compile-Time vs Runtime

The existing `#[cfg(feature = "gssapi")]` blocks in `create_connection()` should be replaced with runtime branching on `self.auth_mode`. The `gssapi` Cargo feature flag remains but only controls whether the `ldap3/gssapi` dependency is compiled in. When built with `gssapi` feature but configured with simple bind env vars, the runtime auth mode takes precedence.

### Security Considerations

- Simple bind transmits credentials in cleartext over LDAP (port 389). This is acceptable for a local lab (Internal virtual switch) but should NOT be used in production.
- Consider adding LDAPS (port 636) support in a future story for secure simple bind.
- The password is held in memory as a `String` - acceptable for a dev/test tool.

### Integration Test Environment

Prerequisites:
- Windows Server VM with AD DS role (Hyper-V, Internal switch)
- Domain populated with BadBlood for realistic test data
- VM IP: `10.0.0.10` (or as configured)
- Test account: a domain user with read access (e.g., `CN=TestBind,CN=Users,DC=lab,DC=dspanel,DC=local`)

```bash
# Run integration tests against the lab
export DSPANEL_LDAP_SERVER=10.0.0.10
export DSPANEL_LDAP_BIND_DN="CN=TestBind,CN=Users,DC=lab,DC=dspanel,DC=local"
export DSPANEL_LDAP_BIND_PASSWORD="TestP@ss123"
cargo test --test ldap_integration -- --nocapture
```
