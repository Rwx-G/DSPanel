# Story 4.6: LDAPS (TLS) Support

## Status: Done

## Story

**As a** system administrator,
**I want** DSPanel to connect to Active Directory via LDAPS (port 636) with TLS encryption,
**so that** sensitive operations like password resets work over simple bind, and all AD communication is encrypted in transit (NFR6).

## Acceptance Criteria

1. DSPanel supports LDAPS connections on port 636 with TLS encryption.
2. The connection protocol (LDAP or LDAPS) is configurable via the `DSPANEL_LDAP_SERVER` environment variable using a `ldaps://` URL scheme, or via a separate `DSPANEL_LDAP_USE_TLS` flag.
3. When LDAPS is configured, all LDAP operations (search, bind, modify) use the encrypted channel.
4. Password reset (`unicodePwd` modification) works over LDAPS with simple bind.
5. Self-signed or internal CA certificates are supported via a configurable CA certificate path (`DSPANEL_LDAP_CA_CERT`), or a flag to skip TLS verification for lab environments (`DSPANEL_LDAP_TLS_SKIP_VERIFY`).
6. GSSAPI (Kerberos) authentication continues to work unchanged (already encrypted).
7. Connection errors due to TLS (certificate validation, handshake failure) produce clear error messages.
8. Unit tests cover TLS configuration parsing and URL scheme detection.
9. Integration tests validate LDAPS connectivity when a TLS-enabled DC is available.

## Dev Notes

### ldap3 TLS Support

The `ldap3` crate supports TLS via the `LdapConnSettings`:

```rust
let settings = LdapConnSettings::new()
    .set_no_tls_verify(true); // For self-signed certs in lab

// LDAPS (implicit TLS on port 636):
let (conn, ldap) = LdapConnAsync::with_settings(
    settings,
    "ldaps://dc.example.com:636"
).await?;

// StartTLS (upgrade on port 389):
let settings = LdapConnSettings::new()
    .set_starttls(true);
```

### Environment Variables

```bash
# Option A: URL scheme detection
export DSPANEL_LDAP_SERVER=ldaps://172.31.72.165

# Option B: Explicit flag
export DSPANEL_LDAP_SERVER=172.31.72.165
export DSPANEL_LDAP_USE_TLS=true

# Optional: Custom CA cert for internal PKI
export DSPANEL_LDAP_CA_CERT=/path/to/ca.pem

# Optional: Skip TLS verification (lab only)
export DSPANEL_LDAP_TLS_SKIP_VERIFY=true
```

### AD Certificate Requirements

For LDAPS to work, the domain controller needs a valid certificate. Options:
- AD CS (Active Directory Certificate Services) - enterprise CA
- Self-signed certificate on the DC
- Third-party CA certificate

For the test lab (Hyper-V), the simplest approach is to install AD CS on the
DC or use `DSPANEL_LDAP_TLS_SKIP_VERIFY=true`.

### Password Reset Requirement

AD requires an encrypted channel to modify `unicodePwd`. Accepted channels:
- LDAPS (port 636) - works with simple bind
- Kerberos (GSSAPI) - already encrypted
- StartTLS on port 389 - also works but less common
