# Story 5.1: Preset Storage and Configuration

## Status: Draft

## Story

**As a** system administrator,
**I want** to configure a shared network path where onboarding/offboarding presets are stored as JSON files,
**so that** preset definitions are centralized, versioned, and accessible to all DSPanel operators.

## Acceptance Criteria

1. A "Presets" section in Settings allows configuring the network share path for preset storage.
2. The configured path is validated at application startup - if unreachable, a warning banner is shown but the app remains functional.
3. Each preset is stored as a separate JSON file (one file per preset) in the configured directory.
4. A JSON schema defines the preset structure and is enforced on load - invalid files are skipped with a logged warning.
5. A `notify crate file watcher` monitors the preset directory for external changes and reloads automatically.
6. `PresetService` trait provides `LoadAllAsync()`, `SaveAsync(preset)`, and `DeleteAsync(presetName)` methods.
7. Error conditions (path not found, permission denied, invalid JSON) display user-friendly messages.
8. Unit tests cover path validation, schema validation, load/save/delete, and file watcher behavior.

## Dev Notes

### Preset JSON Schema

```json
{
  "name": "New Developer Onboarding",
  "description": "Standard setup for new developers joining the team",
  "type": "Onboarding",
  "targetOU": "OU=Developers,OU=Users,DC=contoso,DC=com",
  "groups": [
    "CN=Developers,OU=Groups,DC=contoso,DC=com",
    "CN=VPN-Users,OU=Groups,DC=contoso,DC=com"
  ],
  "attributes": {
    "department": "Engineering",
    "company": "Contoso Ltd",
    "title": "Software Developer"
  }
}
```

### PresetService trait Interface

```typescript
public interface PresetService trait
{
    Task<IReadOnlyList<Preset>> LoadAllAsync();
    Task SaveAsync(Preset preset);
    Task DeleteAsync(string presetName);
    event EventHandler<PresetsChangedEventArgs>? PresetsChanged;
}
```

### File Layout

- `Services/Presets/PresetService.rs` - interface
- `Services/Presets/PresetService.rs` - implementation
- `Services/Presets/PresetSchemaValidator.rs` - JSON validation
- `Models/Preset.rs` - preset model
- `Models/PresetType.rs` - enum (Onboarding, Offboarding)
