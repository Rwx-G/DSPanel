# Story 5.2: Preset Editor UI

## Status: Done

## Story

**As an** AccountOperator or DomainAdmin,
**I want** a visual editor to create, modify, and delete onboarding/offboarding presets,
**so that** I can manage preset definitions without editing JSON files manually.

## Acceptance Criteria

1. A preset list view displays all available presets with name, type, and description.
2. An editor form allows editing preset fields: name, description, type (Onboarding/Offboarding), target OU, groups, and custom attributes.
3. A GroupPicker control allows searching and selecting AD groups to include in the preset.
4. An OUPicker control allows browsing the OU tree to select the target OU.
5. Save validates all required fields and writes the preset JSON via `PresetService trait.SaveAsync()`.
6. Delete prompts for confirmation via `DialogService` trait before removing via `PresetService trait.DeleteAsync()`.
7. The preset editor is gated to AccountOperator+ permission level.
8. Unit tests cover React component logic, validation, permission gating, and save/delete flows.

## Dev Notes

### React component Structure

`PresetManagementComponent` uses React hooks with `React state (useState)` and `event handler` attributes. The GroupPicker and OUPicker are reusable controls that can be embedded in other views (e.g., the onboarding wizard).

### GroupPicker Implementation

The GroupPicker queries `DirectoryProvider trait.SearchAsync()` with an LDAP filter for `(objectClass=group)` combined with the search text. Results are displayed in a dropdown and selected groups are added to an `React state array<GroupInfo>`. Each `GroupInfo` contains `DisplayName` and `DistinguishedName`.

### OUPicker Implementation

The OUPicker calls `DirectoryProvider trait.GetOUTreeAsync()` to load the OU hierarchy. The tree is lazily loaded - child OUs are fetched when a node is expanded. The selected OU's DN is bound to the React component.
