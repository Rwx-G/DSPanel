# Story 5.3: Onboarding Wizard

## Status: Done

## Story

**As an** AccountOperator or DomainAdmin,
**I want** a step-by-step wizard to create new user accounts using preset templates,
**so that** I can onboard employees consistently with all required groups, attributes, and settings applied automatically.

## Acceptance Criteria

1. The wizard has four steps: User Details, Preset Selection, Preview Diff, and Confirm/Execute.
2. User Details step collects first name, last name, display name, and generates a login (sAMAccountName) automatically based on naming convention.
3. A secure password is auto-generated via `PasswordService trait.GenerateAsync()` with option for manual override.
4. Preset Selection step lists available onboarding presets and applies the selected preset's groups, OU, and attributes.
5. Preview Diff step shows all changes that will be applied (OU placement, group memberships, attributes) in a readable diff format.
6. Confirm/Execute step creates the user and applies all settings, showing progress.
7. On completion, a copyable output summary displays the new account details (login, password, OU, groups).
8. Every onboarding operation is logged via `AuditService` trait.
9. The wizard is gated to AccountOperator+ permission level.
10. Unit tests cover each wizard step, navigation, execution, and audit logging.

## Dev Notes

### Login Generation

The auto-generated login follows a configurable pattern. Default: `{first initial}{last name}` lowercased (e.g., "John Smith" becomes "jsmith"). If the login already exists, a numeric suffix is appended (jsmith1, jsmith2). Uniqueness is checked via `DirectoryProvider trait.SearchAsync()`.

### Execution Order

1. Create user object in target OU with basic attributes
2. Set password (unicodePwd)
3. Enable the account (userAccountControl)
4. Add to each group sequentially
5. Set additional attributes from preset

If any step fails after the user is created, the wizard offers to delete the partially created user (rollback) or keep it for manual fixup.

### Rollback Strategy

On failure, the wizard calls `DirectoryProvider trait.DeleteObjectAsync(createdUserDn)` to clean up. If the delete also fails, the error is logged and the user is informed of the orphaned object.
