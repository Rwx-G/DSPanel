# Story 5.4: Offboarding Workflow

## Status: Done

## Story

**As an** AccountOperator or DomainAdmin,
**I want** a guided offboarding workflow triggered from a user's context menu,
**so that** I can consistently disable accounts, remove memberships, and apply departure policies with full audit trailing.

## Acceptance Criteria

1. A "Start Offboarding" option is available in the user lookup context menu, gated to AccountOperator+.
2. The workflow steps are: Confirm User, Select Actions, Preview Changes, Execute.
3. Actions are individually toggleable: disable account, remove all group memberships, set mail forwarding, move to Disabled OU, set random password.
4. A dry-run preview shows exactly what changes will be applied before execution.
5. On completion, a copyable output summary lists all changes made.
6. Every offboarding operation is logged via `AuditService` trait with full details.
7. Unit tests cover action toggles, dry-run, execution, output, and audit.

## Dev Notes

### Action Details

| Action              | AD Operation                | Attribute/Method                                                                         |
| ------------------- | --------------------------- | ---------------------------------------------------------------------------------------- |
| Disable account     | Modify `userAccountControl` | Set `ACCOUNTDISABLE` flag (0x2)                                                          |
| Remove groups       | Remove from all groups      | `DirectoryProvider trait.RemoveFromGroupAsync()` for each group                          |
| Set forwarding      | Modify `altRecipient`       | Set to forwarding target DN                                                              |
| Move to Disabled OU | LDAP ModifyDN               | `DirectoryProvider trait.MoveObjectAsync(userDn, disabledOU)`                            |
| Set random password | Modify `unicodePwd`         | `PasswordService trait.GenerateAsync()` + `DirectoryProvider trait.ResetPasswordAsync()` |

### Group Removal

When removing groups, the workflow first fetches all current memberships via `DirectoryProvider trait.GetGroupMembershipsAsync()`, then removes the user from each group except "Domain Users" (which cannot be removed as it is the primary group).

### Context Menu Integration

The "Start Offboarding" menu item is added to the user lookup result context menu. It passes the selected user's DN to the `OffboardingComponent` constructor.
