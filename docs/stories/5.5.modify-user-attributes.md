# Story 5.5: Modify User Attributes

## Status: Draft

## Story

**As an** AccountOperator,
**I want** to edit user attributes directly from the User Lookup detail panel,
**so that** I can correct or update user information (name, email, department, phone, etc.) without switching to another tool.

## Acceptance Criteria

1. Editable fields are displayed with an inline edit affordance (pencil icon or click-to-edit) in the User Detail PropertyGrid.
2. Only users with AccountOperator+ permission see the edit controls.
3. Editable attributes include: `displayName`, `givenName`, `sn`, `mail`, `department`, `title`, `telephoneNumber`, `manager`, `description`, and extension attributes (1-15).
4. The Advanced Attributes section supports editing any attribute value by clicking on it (for AccountOperator+).
5. Changes are validated before submission (non-empty DN fields, valid email format for `mail`).
6. A confirmation dialog summarizes all pending changes before execution (dry-run preview).
7. A snapshot is captured before any modification for audit/rollback purposes.
8. Each attribute modification is logged via `AuditService`.
9. The user detail view refreshes automatically after successful modification.
10. Errors are displayed as user-friendly notifications (not raw JSON).
11. Unit tests cover permission gating, validation, confirmation flow, and error handling.

### Editable Attributes List

Standard user attributes that are commonly edited:
- `displayName`, `givenName`, `sn` (identity)
- `mail`, `telephoneNumber` (contact)
- `department`, `title`, `company` (organization)
- `manager` (hierarchy - DN picker)
- `description` (free text)
- `extensionAttribute1` through `extensionAttribute15` (custom)

Attributes that should NOT be editable via this interface:
- `sAMAccountName`, `userPrincipalName` (identity - requires rename operation)
- `userAccountControl` (flags - managed by dedicated UI)
- `memberOf` (groups - managed by group management)
- `distinguishedName` (identity - requires move operation)
- `pwdLastSet`, `lockoutTime` (managed by dedicated actions)

### Security Considerations

- Permission check on every modify call (backend enforced, not just UI)
- Snapshot before modify for rollback capability
- Audit trail for compliance
- Input validation to prevent LDAP injection via attribute values
