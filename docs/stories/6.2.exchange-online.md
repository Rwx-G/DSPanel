# Story 6.2: Exchange Online Diagnostics (Graph)

## Status: Draft

## Story

**As a** HelpDesk operator,
**I want** to see Exchange Online mailbox details for cloud-hosted mailboxes via Microsoft Graph,
**so that** I can troubleshoot mail issues for users whose mailboxes are in Microsoft 365.

## Acceptance Criteria

1. When Azure AD/Graph integration is configured and a user has an Exchange Online mailbox, an "Exchange Online" panel appears in the user detail view.
2. The panel displays: email aliases, forwarding rules, mailbox quota usage, and delegate permissions.
3. Settings provide fields for Azure tenant ID, application (client) ID, and authentication method.
4. The panel is hidden when Graph integration is not configured or the user has no Exchange Online mailbox.
5. All Exchange Online fields are read-only (display only).
6. Required Graph API permissions are documented: `Mail.Read`, `User.Read.All`.
7. Unit tests cover Graph API integration, model mapping, settings validation, and conditional visibility.

## Dev Notes

### Microsoft Graph SDK Usage

```typescript
var credential = new OAuth2 client credentials(tenantId, clientId, clientSecret);
// or
var credential = new OAuth2 browser flow(new OAuth2 browser flowOptions
{
    TenantId = tenantId,
    ClientId = clientId
});

var graphClient = new Microsoft Graph REST API client(credential, new[] { "https://graph.microsoft.com/.default" });

// Mailbox settings
var settings = await graphClient.Users[userId].MailboxSettings.GetAsync();

// Mail aliases from user profile
var user = await graphClient.Users[userId].GetAsync(config =>
    config.QueryParameters.Select = new[] { "proxyAddresses", "mail" });
```

### Required Azure AD App Registration

- Application type: Public client (for Interactive Browser) or Confidential client (for Client Secret)
- API permissions: `Mail.Read` (Delegated or Application), `User.Read.All` (Application)
- Admin consent required for Application permissions

### Authentication Methods

| Method | Use Case | Credential Class |
|--------|----------|-----------------|
| Interactive Browser | Per-user, operator signs in | `OAuth2 browser flow` |
| Client Secret | Unattended, service account | `OAuth2 client credentials` |
