# Story 7.1: Move Objects Between OUs

## Status: Review

## Story

**As an** AccountOperator or DomainAdmin,
**I want** to move AD objects (users, computers, groups) between Organizational Units via a visual OU picker,
**so that** I can reorganize directory structure without using ADUC or PowerShell.

## Acceptance Criteria

1. A "Move to OU" option is available in the object context menu for users, computers, and groups.
2. An OU picker dialog displays the OU tree for selecting the destination.
3. Single-object and bulk-move (multiple selected objects) are both supported.
4. A dry-run preview shows the current and target OU for each object before execution.
5. The operation is gated to AccountOperator+ permission level.
6. Every move operation is logged via `AuditService` trait with source and destination OUs.
7. Unit tests cover single move, bulk move, dry-run, permission gating, and audit logging.

## Dev Notes

### LDAP ModifyDN Operation

Moving an object in LDAP uses the `LDAP modify DN request`:

```typescript
var request = new LDAP modify DN request(
    objectDn,           // current DN
    targetOuDn,         // new superior (target OU)
    $"CN={objectCn}"    // new RDN (keep same CN)
);
var response = (LDAP modify DN response)connection.SendRequest(request);
```

### Bulk Move Error Handling

Bulk moves are processed sequentially. Each move is attempted independently. If one fails, the error is recorded and the next object is processed. The final result shows a summary: X succeeded, Y failed, with details for each failure.
