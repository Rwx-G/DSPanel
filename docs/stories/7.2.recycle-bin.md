# Story 7.2: AD Recycle Bin

## Status: Done

## Story

**As a** DomainAdmin,
**I want** to browse and restore deleted Active Directory objects from the AD Recycle Bin,
**so that** I can recover accidentally deleted users, groups, or computers without resorting to authoritative restores.

## Acceptance Criteria

1. A "Recycle Bin" view lists deleted objects with name, object type, deletion date, and original OU.
2. Search and filter are available to find deleted objects by name or type.
3. A restore action recovers the object to its original OU, or to a user-specified OU if the original no longer exists.
4. A warning is displayed if the AD Recycle Bin feature is not enabled on the domain.
5. The Recycle Bin view is gated to DomainAdmin permission level.
6. Every restore operation is logged via `AuditService` trait.
7. Unit tests cover listing, search, restore, feature detection, and permission gating.

## Dev Notes

### LDAP Query for Deleted Objects

The Deleted Objects container requires the `ShowDeletedControl` to be included in the search request:

```typescript
var searchRequest = new LDAP search request(
    $"CN=Deleted Objects,{domainDn}",
    "(isDeleted=TRUE)",
    SearchScope.OneLevel,
    "cn", "objectClass", "lastKnownParent", "whenChanged", "distinguishedName"
);
searchRequest.Controls.Add(new DirectoryControl("1.2.840.113556.1.4.417", null, true, true));
```

### Restore Operation

Restoring a deleted object involves:

1. Remove the `isDeleted` attribute
2. Move the object to the target OU via `LDAP modify DN request`
3. The object's name has `\0ADEL:<guid>` appended - strip this suffix when restoring

### AD Recycle Bin Feature GUID

The Recycle Bin optional feature GUID is `766ddcd8-acd0-445e-f3b9-a7f9b6744f2a`. Check if this GUID appears in `msDS-EnabledFeature` on the Partitions container.
