# Story 7.3: Contact and Printer Management

## Status: Done

## Story

**As an** administrator,
**I want** to view, create, edit, and delete AD contacts and printers from DSPanel,
**so that** I can manage all directory object types from a single tool.

## Acceptance Criteria

1. Contacts appear in search results alongside users, computers, and groups, distinguished by a contact icon.
2. A contact detail view shows all relevant attributes (display name, email, phone, company, address).
3. CRUD operations on contacts are available for AccountOperator+ users.
4. Printers appear in search results with a printer icon and show name, location, server, and share path.
5. Printer detail view displays all printer attributes in read-only mode for all users.
6. Printer management (create, edit, delete) is gated to DomainAdmin.
7. All create, update, and delete operations are logged via `AuditService` trait.
8. Unit tests cover search integration, CRUD operations, permission gating, and audit logging.

## Dev Notes

### LDAP Object Classes

| Object  | LDAP Filter                | Key Attributes                                                                                                                                       |
| ------- | -------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------- |
| Contact | `(objectClass=contact)`    | `cn`, `displayName`, `givenName`, `sn`, `mail`, `telephoneNumber`, `mobile`, `company`, `department`, `streetAddress`, `l`, `st`, `postalCode`, `co` |
| Printer | `(objectClass=printQueue)` | `printerName`, `location`, `serverName`, `uNCName`, `driverName`, `portName`, `description`                                                          |

### Search Integration

The existing search query is extended with additional objectClass conditions. The search result model gains a `ResultType` enum value for `Contact` and `Printer`, which drives icon selection in the UI.
