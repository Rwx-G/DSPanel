# Story 7.4: User Thumbnail Photo

## Status: Done

## Story

**As an** AccountOperator or DomainAdmin,
**I want** to view, upload, and remove user thumbnail photos in Active Directory,
**so that** user profile images are consistent across AD-integrated applications (Outlook, Teams, SharePoint).

## Acceptance Criteria

1. The user detail view displays the user's `thumbnailPhoto` attribute as an avatar image, or a placeholder icon if none is set.
2. An "Upload Photo" button allows selecting a JPG or PNG file from disk.
3. Uploaded images are automatically resized to 96x96 pixels before writing to AD.
4. A "Remove Photo" button clears the `thumbnailPhoto` attribute.
5. Photo modification is gated to AccountOperator+ permission level.
6. All photo changes are logged via `AuditService` trait.
7. Unit tests cover photo display, upload/resize, removal, permission gating, and audit logging.

## Dev Notes

### Image Processing

The `thumbnailPhoto` attribute in AD is limited to approximately 100KB. Images should be resized to 96x96 pixels and compressed as JPEG to stay within this limit. The resize operation should maintain the aspect ratio and center-crop if needed.

```typescript
// Using image crate (Windows-only, acceptable for React or Tauri)
using var original = Image.FromFile(filePath);
using var resized = new Bitmap(96, 96);
using var graphics = Graphics.FromImage(resized);
graphics.InterpolationMode = InterpolationMode.HighQualityBicubic;
graphics.DrawImage(original, 0, 0, 96, 96);

using var ms = new MemoryStream();
resized.Save(ms, ImageFormat.Jpeg);
byte[] photoBytes = ms.ToArray();
```

### AD Attribute

The `thumbnailPhoto` attribute is a single-valued binary attribute. Maximum recommended size is 10KB for best compatibility across AD-integrated applications. The 96x96 JPEG typically produces files under 5KB.
