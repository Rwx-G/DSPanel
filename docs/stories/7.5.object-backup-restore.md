# Story 7.5: Object Backup and Restore

## Status: Draft

## Story

**As a** DomainAdmin,
**I want** automatic snapshots of AD objects taken before every write operation, with the ability to browse history and restore previous states,
**so that** accidental or incorrect changes can be rolled back safely.

## Acceptance Criteria

1. Before any write operation (modify, move, delete), DSPanel automatically captures a snapshot of the target object's current attributes.
2. Snapshots are stored locally in a SQLite database with timestamp, object DN, operation type, and serialized attributes.
3. A "History" tab in the object detail view shows all snapshots for that object, sorted by date.
4. Selecting a snapshot shows a diff between the snapshot and the current state.
5. A "Restore" action applies the snapshot's attributes back to the object, with a dry-run preview.
6. Snapshot retention is configurable (default 30 days), with automatic cleanup of expired snapshots.
7. Restore is gated to DomainAdmin permission level.
8. Unit tests cover snapshot capture, storage, diff, restore, retention, and permission gating.

## Dev Notes

### SQLite Schema

```sql
CREATE TABLE Snapshots (
    Id INTEGER PRIMARY KEY AUTOINCREMENT,
    ObjectDn TEXT NOT NULL,
    OperationType TEXT NOT NULL,
    Timestamp TEXT NOT NULL,
    OperatorDn TEXT NOT NULL,
    Attributes TEXT NOT NULL,
    CreatedAt TEXT NOT NULL DEFAULT (datetime('now'))
);

CREATE INDEX IX_Snapshots_ObjectDn ON Snapshots(ObjectDn);
CREATE INDEX IX_Snapshots_Timestamp ON Snapshots(Timestamp);
```

### rusqlite Query Examples

```typescript
// Insert snapshot
await connection.ExecuteAsync(
    "INSERT INTO Snapshots (ObjectDn, OperationType, Timestamp, OperatorDn, Attributes) VALUES (@ObjectDn, @OperationType, @Timestamp, @OperatorDn, @Attributes)",
    snapshot,
);

// Get history
var snapshots = await connection.QueryAsync<ObjectSnapshot>(
    "SELECT * FROM Snapshots WHERE ObjectDn = @ObjectDn ORDER BY Timestamp DESC",
    new { ObjectDn = objectDn }(),
);
```

### Read-Only Attributes (Skip on Restore)

These attributes cannot be modified and must be excluded from restore operations:

- `objectGuid`, `objectSid`, `whenCreated`, `whenChanged`
- `uSNCreated`, `uSNChanged`, `objectCategory`
- `distinguishedName`, `cn` (use ModifyDN for moves instead)
