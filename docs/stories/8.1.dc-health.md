# Story 8.1: Domain Controller Health Checks

## Status: Done

## Story

**As a** DomainAdmin,
**I want** a dashboard showing the health status of all domain controllers in my environment,
**so that** I can quickly identify infrastructure issues before they impact users.

## Acceptance Criteria

1. An "Infrastructure Health" view displays status cards for each domain controller in the domain.
2. Each DC card shows: DNS resolution status, AD services status, SYSVOL accessibility, disk space, and LDAP response time.
3. Status indicators use color coding: green (healthy), yellow (warning), red (critical).
4. Auto-refresh is configurable (default: 60 seconds) with a manual refresh button.
5. Clicking a DC card opens a detail panel with full health check results.
6. The Infrastructure Health view is gated to DomainAdmin permission level.
7. Unit tests cover health check logic, status computation, and auto-refresh behavior.

## Dev Notes

### WMI Queries

WMI queries are performed using `windows` crate WMI bindings (Windows-only) or CIM via `windows crate CIM bindings`:

```typescript
// Service status
var searcher = new WMI query via windows crate(
    $"\\\\{dcHostname}\\root\\cimv2",
    "SELECT Name, State FROM Win32_Service WHERE Name='NTDS' OR Name='DNS' OR Name='Netlogon' OR Name='KDC'");

// Disk space
var diskSearcher = new WMI query via windows crate(
    $"\\\\{dcHostname}\\root\\cimv2",
    "SELECT DeviceID, FreeSpace, Size FROM Win32_LogicalDisk WHERE DeviceID='C:'");
```

### LDAP Ping

Measure LDAP response time by performing a simple RootDSE bind to the specific DC:

```typescript
var stopwatch = std::time::Instant.StartNew();
var connection = new ldap3 connection(dcHostname);
connection.Bind();
stopwatch.Stop();
var responseTimeMs = stopwatch.ElapsedMilliseconds;
```

### Health Check Thresholds

| Check         | Green       | Yellow    | Red                    |
| ------------- | ----------- | --------- | ---------------------- |
| Disk free     | > 20%       | 10-20%    | < 10%                  |
| LDAP response | < 100ms     | 100-500ms | > 500ms or unreachable |
| Services      | All running | N/A       | Any stopped            |
| SYSVOL        | Accessible  | N/A       | Inaccessible           |
| DNS           | Resolves    | N/A       | Fails                  |
