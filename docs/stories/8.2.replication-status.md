# Story 8.2: AD Replication Status

## Status: Done

## Story

**As a** DomainAdmin,
**I want** to view the status of all AD replication partnerships and force replication when needed,
**so that** I can monitor replication health and resolve sync issues proactively.

## Acceptance Criteria

1. A "Replication" view shows all replication partnerships with source DC, target DC, last successful sync, status, and error count.
2. Failed replication partnerships are highlighted in red with error details.
3. Replication latency (time since last successful sync) is displayed for each partnership.
4. A "Force Replication" button triggers replication sync for a selected partnership, with a confirmation dialog.
5. The view auto-refreshes at a configurable interval (default: 120 seconds).
6. The Replication view is gated to DomainAdmin permission level.
7. Unit tests cover partnership listing, error highlighting, force replication, and permission gating.

## Dev Notes

### Replication Data Sources

Option A - LDAP query of `msDS-ReplNeighbor`:

```typescript
var request = new LDAP search request(
    "", // RootDSE
    "(objectClass=*)",
    SearchScope.Base,
    "msDS-ReplNeighbor"
);
```

Option B - Parse `repadmin /showrepl /csv`:

```typescript
var process = open crate(new Command builder
{
    FileName = "repadmin",
    Arguments = "/showrepl * /csv",
    RedirectStandardOutput = true,
    UseShellExecute = false
});
var csvOutput = await process.StandardOutput.ReadToEndAsync();
```

Option B is simpler and provides a comprehensive view across all DCs.

### DsReplicaSync

Force replication can be triggered via:

- P/Invoke to `DsReplicaSync` from `ntdsapi.dll`
- Running `repadmin /replicate <destDC> <sourceDC> <namingContext>`

The `repadmin` approach is simpler and does not require P/Invoke declarations.
