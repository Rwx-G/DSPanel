# Story 8.3: DNS and Kerberos Validation

## Status: Draft

## Story

**As a** DomainAdmin,
**I want** to validate critical DNS SRV records and check Kerberos clock skew across domain controllers,
**so that** I can detect and fix AD infrastructure issues that cause authentication or service discovery failures.

## Acceptance Criteria

1. The validation view checks `_ldap._tcp`, `_kerberos._tcp`, `_gc._tcp`, and `_kpasswd._tcp` SRV records for the domain.
2. Each record shows expected vs. actual results with pass/fail status.
3. Kerberos clock skew is checked between each DC and the local machine, with a configurable threshold (default: 5 minutes).
4. DCs exceeding the clock skew threshold are flagged with a warning.
5. Results are exportable to CSV or text file.
6. The validation view is gated to DomainAdmin permission level.
7. Unit tests cover DNS record validation, clock skew detection, threshold logic, and export.

## Dev Notes

### DNS SRV Record Queries

Using `hickory-dns` cargo or npm package (if approved) or `hickory-dns crate`:

```typescript
// Using hickory-dns
var lookup = new LookupClient();
var result = await lookup.QueryAsync($"_ldap._tcp.{domainName}", QueryType.SRV);
var srvRecords = result.Answers.SrvRecords();
```

If `hickory-dns` is not available, use P/Invoke to `DnsQuery` from `dnsapi.dll`.

### Clock Skew via LDAP RootDSE

The simplest approach to check DC time is reading `currentTime` from each DC's RootDSE:

```typescript
var connection = new ldap3 connection(dcHostname);
var request = new LDAP search request("", "(objectClass=*)", SearchScope.Base, "currentTime");
var response = (LDAP search response)connection.SendRequest(request);
var dcTime = DateTime.Parse(response.Entries[0].Attributes["currentTime"][0].ToString());
var skew = Math.Abs((dcTime - DateTime.UtcNow).TotalSeconds);
```

### Critical SRV Records

| Record | Purpose |
|--------|---------|
| `_ldap._tcp.<domain>` | LDAP service discovery |
| `_kerberos._tcp.<domain>` | Kerberos KDC discovery |
| `_gc._tcp.<forest>` | Global Catalog discovery |
| `_kpasswd._tcp.<domain>` | Kerberos password change |
| `_ldap._tcp.dc._msdcs.<domain>` | DC-specific LDAP |
