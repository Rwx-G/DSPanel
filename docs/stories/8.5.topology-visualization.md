# Story 8.5: AD Topology Visualization

## Status: Done

## Story

**As a** DomainAdmin,
**I want** a visual graph showing AD sites, domain controllers, and replication links,
**so that** I can understand the replication topology at a glance and identify connectivity issues.

## Acceptance Criteria

1. A topology graph displays sites as regions/containers, DCs as nodes within their sites, and replication links as edges between DCs.
2. Replication link edges are color-coded by status: green (healthy), yellow (warning), red (failed).
3. Site link costs and replication intervals are displayed on edges connecting sites.
4. Nodes and edges are interactive - clicking opens a detail panel with full information.
5. The graph supports zoom and pan for large topologies.
6. The topology can be exported as a PNG image.
7. The topology view is gated to DomainAdmin permission level.
8. Unit tests cover data model construction, status computation, and export logic.

## Dev Notes

### AD Sites and Services LDAP Structure

```
CN=Sites,CN=Configuration,DC=contoso,DC=com
  CN=Default-First-Site-Name          (site)
    CN=Servers
      CN=DC1                          (server)
        CN=NTDS Settings              (nTDSDSA - has connection objects)
  CN=Inter-Site Transports
    CN=IP
      CN=DEFAULTIPSITELINK             (siteLink)
```

### Key LDAP Attributes

| Object          | Attributes                                               |
| --------------- | -------------------------------------------------------- |
| Site            | `cn`, `location`, `siteObjectBL`                         |
| Server          | `cn`, `dNSHostName`, `serverReference`                   |
| NTDS Settings   | `objectGUID`, `options` (isGC flag), `msDS-HasMasterNCs` |
| NTDS Connection | `fromServer`, `schedule`, `options`                      |
| Site Link       | `siteList`, `cost`, `replInterval`                       |

### Graph Layout

For automatic layout, consider:

- **Force-directed layout**: Good for general graphs, may not respect site grouping
- **Hierarchical layout**: Sites as containers with DCs arranged inside
- **Manual positioning**: Sites placed in a grid, DCs within each site cell

The hierarchical/container approach is recommended to visually group DCs by site.

### PNG Export

const canvas = canvasRef.current;

    if (blob) {
        const url = URL.createObjectURL(blob);
        const link = document.createElement("a");

        link.click();
