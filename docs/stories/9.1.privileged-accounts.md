# Story 9.1: Privileged Accounts Dashboard

## Status: Draft

## Story

**As a** DomainAdmin,
**I want** a dashboard listing all privileged accounts with health indicators and alerts,
**so that** I can quickly identify high-risk accounts and take corrective action.

## Acceptance Criteria

1. The dashboard lists members of Domain Admins, Enterprise Admins, Schema Admins, and any additional configurable groups.
2. Each account row shows: SAMAccountName, display name, last logon, password age, password expiry date, and enabled/disabled status.
3. Alerts are raised for: password older than 90 days, password set to never expire, account never logged on, and disabled accounts still in a privileged group.
4. Each alert has a severity level (Critical, High, Medium, Info) displayed with color coding.
5. The dashboard provides CSV and PDF export of the full account list including alert details.
6. The view is gated to DomainAdmin permission level only.
7. Unit tests cover the ViewModel, alert computation logic, and permission gating.

## Dev Notes

### Privileged Group Resolution

The default privileged groups are identified by well-known SIDs: Domain Admins (S-1-5-21-...-512), Enterprise Admins (S-1-5-21-...-519), Schema Admins (S-1-5-21-...-518). Additional groups are configurable via settings and stored as distinguished names.

### Alert Severity Levels

| Condition | Severity |
|-----------|----------|
| Password > 90 days old | Critical |
| Password set to never expire | High |
| Disabled account in privileged group | High |
| Account never logged on | Medium |
