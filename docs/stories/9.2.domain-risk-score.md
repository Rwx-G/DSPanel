# Story 9.2: Domain Risk Score

## Status: Draft

## Story

**As a** DomainAdmin,
**I want** a composite risk score for my domain based on weighted security factors,
**so that** I can measure overall AD health and track improvements over time.

## Acceptance Criteria

1. A weighted score from 0 (worst) to 100 (best) is computed from: privileged account hygiene, password policy strength, stale account ratio, and dangerous configuration flags.
2. A gauge visualization displays the score with color zones (red 0-40, orange 41-70, green 71-100).
3. A factor breakdown panel shows each factor's individual score, weight, and a plain-text explanation.
4. Actionable recommendations are generated for each factor scoring below threshold.
5. A 30-day trend chart shows historical scores stored locally in SQLite.
6. Factor weights are configurable by the administrator in settings.
7. Unit tests cover the risk calculator, each factor's scoring logic, and historical storage.

### Factor Weights (Defaults)

| Factor | Default Weight |
|--------|---------------|
| Privileged account hygiene | 30% |
| Password policy strength | 25% |
| Stale account ratio | 25% |
| Dangerous configurations | 20% |

### SQLite Schema

```sql
CREATE TABLE risk_scores (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    date TEXT NOT NULL UNIQUE,
    total_score REAL NOT NULL,
    factors_json TEXT NOT NULL
);
```
