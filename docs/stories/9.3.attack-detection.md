# Story 9.3: AD Attack Detection

## Status: Review

## Story

**As a** DomainAdmin,
**I want** DSPanel to detect common AD attack patterns from event logs,
**so that** I am alerted to potential Golden Ticket, DCSync, DCShadow, and abnormal Kerberos activity.

## Acceptance Criteria

1. The attack detector analyzes Windows Security event logs for events 4768, 4769, 4662, and other relevant IDs.
2. Detection algorithms identify patterns consistent with Golden Ticket, DCSync, DCShadow, and abnormal Kerberos requests.
3. Alerts are displayed in a dedicated dashboard panel with severity, timestamp, source DC, and description.
4. Each alert includes a recommended response action (e.g., "Reset KRBTGT password twice").
5. Detection thresholds are configurable (e.g., maximum ticket lifetime deviation, replication request frequency).
6. The view is gated to DomainAdmin permission level only.
7. Unit tests cover each detection algorithm, alert creation, and threshold configuration.

## Dev Notes

### Event IDs Reference

| Event ID | Description                       | Attack Relevance                             |
| -------- | --------------------------------- | -------------------------------------------- |
| 4768     | Kerberos TGT requested            | Golden Ticket (abnormal lifetime/encryption) |
| 4769     | Kerberos TGS requested            | Abnormal Kerberos activity                   |
| 4662     | Directory service object accessed | DCSync (replication rights)                  |
| 4742     | Computer account changed          | DCShadow (rogue DC)                          |

### Detection Approach

Event log analysis is performed on-demand (not real-time). The detector queries the last N hours of events (configurable, default 24h) from the connected DC. For multi-DC environments, each DC's logs should be queried.
