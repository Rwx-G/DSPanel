# Story 9.4: Privilege Escalation Path Visualization

## Status: Draft

## Story

**As a** DomainAdmin,
**I want** a graph visualization showing possible privilege escalation paths from standard accounts to privileged groups,
**so that** I can identify and remediate unintended access paths in my domain.

## Acceptance Criteria

1. A directed graph displays nodes (users, groups) and edges (membership, ownership, delegation) representing paths to privileged groups.
2. The graph considers direct membership, nested group membership, object ownership, and delegated permissions.
3. Critical escalation paths (shortest paths to Domain Admins) are visually highlighted in red.
4. Clicking a node or edge shows details (object properties, relationship type, how the path was derived).
5. The graph supports zoom, pan, and layout adjustment for large datasets.
6. The view is gated to DomainAdmin permission level only.
7. Unit tests cover the graph-building logic, path-finding algorithm, and edge-type resolution.

## Tasks / Subtasks

- [ ] Task 1: Build group membership graph (AC: #1, #2)
  - [ ] Create `EscalationGraphBuilder` service
  - [ ] Query all group memberships (direct and nested) via `IDirectoryProvider`
  - [ ] Query object ownership (nTSecurityDescriptor owner field)
  - [ ] Query delegated permissions (ACEs granting write/modify on privileged groups)
  - [ ] Build in-memory directed graph structure
- [ ] Task 2: Implement path-finding algorithm (AC: #3)
  - [ ] Use BFS to find shortest paths from any node to privileged group nodes
  - [ ] Score paths by risk (shorter = higher risk, ownership/delegation edges = higher weight)
  - [ ] Mark critical paths (direct or short paths to Domain Admins / Enterprise Admins)
- [ ] Task 3: Consider ownership and delegation edges (AC: #2)
  - [ ] Parse nTSecurityDescriptor to extract owner SID
  - [ ] Parse ACEs for write-member, write-owner, GenericAll on group objects
  - [ ] Create typed edges: Membership, Ownership, Delegation
- [ ] Task 4: Create graph visualization (AC: #1, #5)
  - [ ] WPF Canvas or third-party graph layout (force-directed or hierarchical)
  - [ ] Node shapes: circles for users, squares for groups, highlighted for privileged
  - [ ] Edge styles: solid for membership, dashed for ownership, dotted for delegation
  - [ ] Zoom, pan, and auto-layout controls
- [ ] Task 5: Highlight critical paths (AC: #3)
  - [ ] Draw critical path edges in red with increased thickness
  - [ ] Summary panel listing critical paths with hop count
- [ ] Task 6: Interactive details (AC: #4)
  - [ ] Click node to show properties panel (DN, type, group scope, member count)
  - [ ] Click edge to show relationship details (type, source attribute, ACE details)
- [ ] Task 7: Permission gating (AC: #6)
  - [ ] Gate navigation to DomainAdmin level
- [ ] Task 8: Write tests (AC: #7)
  - [ ] Test graph building with known membership structures
  - [ ] Test BFS path finding returns correct shortest paths
  - [ ] Test ownership and delegation edges are correctly identified
  - [ ] Test critical path detection highlights correct paths

## Dev Notes

### Graph Data Structure

```csharp
public class EscalationGraph
{
    public Dictionary<string, GraphNode> Nodes { get; }
    public List<GraphEdge> Edges { get; }
}

public class GraphNode
{
    public string Dn { get; set; }
    public string DisplayName { get; set; }
    public NodeType Type { get; set; } // User, Group
    public bool IsPrivileged { get; set; }
}

public class GraphEdge
{
    public string SourceDn { get; set; }
    public string TargetDn { get; set; }
    public EdgeType Type { get; set; } // Membership, Ownership, Delegation
}
```

### Performance Considerations

For large domains, the full graph can be very large. The builder should support scoped queries (e.g., only paths leading to a selected privileged group) and lazy loading of subtrees. Consider caching the graph in memory and refreshing on demand.
